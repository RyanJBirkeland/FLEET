# MCP Server

Anthropic MCP (Model Context Protocol) server for exposing FLEET capabilities to Claude Code and other clients.
Source: `src/main/mcp-server/`

| Module | Purpose | Key Exports |
|--------|---------|-------------|
| [`auth.ts`](./auth.md) | Bearer-token authentication middleware with constant-time comparison for HTTP requests. `parseBearerToken` is exported as a named helper so the whitespace tolerance is explicit and independently testable. | `checkBearerAuth`, `parseBearerToken`, `AuthResult` |
| `close-quietly.ts` | Awaits a `Closable.close()` and swallows the failure — logging via `logError` with full stack. Shared between shutdown paths that must not abort on one failing close. | `closeQuietly`, `Closable` |
| [`auth-rate-limit.ts`](./auth-rate-limit.md) | Progressive-delay rate limit for repeated 401s. Per-remote-address counter; delays kick in after 10 failures in a 60s window, escalate exponentially to a 5s cap, and clear on successful auth. Emits a one-shot `mcp.auth.brute-force-suspected` warn log at the threshold. Wired into `transport.ts` as the `rateLimit` parameter of `createTransportHandler` — defaults to a per-handler `createAuthRateLimit({ logger })` when the composition root doesn't inject one. | `createAuthRateLimit`, `AuthRateLimit`, `computeDelayMs`, `BRUTE_FORCE_THRESHOLD`, `WINDOW_MS`, `INITIAL_DELAY_MS`, `MAX_DELAY_MS` |
| [`errors.ts`](./errors.md) | JSON-RPC error mapping for service exceptions and domain rule violations; `toJsonRpcError` enriches zod field errors with `.describe()` text when a schema is supplied (or when `parseToolArgs` wraps the failure in `McpZodError`) and logs unknown throws via an optional logger; `writeJsonRpcError` writes a full JSON-RPC 2.0 envelope to an `http.ServerResponse` | `toJsonRpcError`, `writeJsonRpcError`, `parseToolArgs`, `McpDomainError`, `McpZodError`, `McpErrorCode`, `JsonRpcErrorBody`, `JSON_RPC_*` code constants |
| [`safe-tool-handler.ts`](./safe-tool-handler.md) | Uniform error-logging wrapper for MCP tool callbacks — the MCP analog of `safeHandle()`. Logs unknown throws with the tool name before they propagate, so failures show up in `~/.fleet/fleet.log`. `wrapServerWithSafeToolHandlers` intercepts both `McpServer.tool(...)` and `McpServer.registerTool(...)` so no call site has to opt in. **T-29:** `wrapRegistrationMethod` uses a file-local `McpServerRegistrar` interface (single `as unknown` cast at entry) — all four inline `as any` casts removed. | `safeToolHandler`, `wrapServerWithSafeToolHandlers` |
| [`schemas.ts`](./schemas.md) | Zod schemas for every MCP tool argument shape; length-capped fields carry `.describe()` text for client discoverability and enriched error messages. Also exports pagination defaults used by `tools/tasks.ts` — `TASK_LIST_DEFAULT_LIMIT` / `TASK_LIST_DEFAULT_OFFSET` back `tasks.list`, and `TASK_HISTORY_DEFAULT_LIMIT` / `TASK_HISTORY_MAX_WINDOW` cap `tasks.history` so unbounded offsets can't reach the DB. `TaskCancelSchema` includes an optional `force` boolean required for cancelling `review`/`done` tasks. | `TaskWriteFieldsSchema`, `TaskCreateSchema`, `TaskUpdateSchema`, `EpicWriteFieldsSchema`, `TASK_LIST_DEFAULT_LIMIT`, `TASK_LIST_DEFAULT_OFFSET`, `TASK_HISTORY_DEFAULT_LIMIT`, `TASK_HISTORY_MAX_WINDOW`, ... |
| [`settings-events.ts`](./settings-events.md) | In-process event bus for settings-change notifications — allows main-process modules to hot-respond to config changes without IPC round-trips | `emitSettingChanged`, `onSettingChanged`, `SettingChangedEvent` |
| [`repos-cache.ts`](./repos-cache.md) | Memoized reader for the `repos` setting. Caches the parsed `RepoConfig[]` so hot calls to `meta.repos` don't re-hit SQLite and JSON.parse on every request; invalidates on `settings-changed` events for `repos`. | `createReposCache`, `ReposCache`, `ReposCacheOptions` |
| `test-setup.ts` | Test precondition seeding — ensures integration tests fail loudly when required repo configs are missing instead of silently skipping | `seedFleetRepo` |
| [`token-store.ts`](./token-store.md) | Persistent bearer token storage at `~/.fleet/mcp-token` — exclusive-create (`fs.open('wx', 0o600)`) with parent dir locked to `0700`; returns `{ token, created, path }` so callers can surface freshly-minted tokens. Optional logger records non-ENOENT read errors, corrupt-token regeneration, and file-mode drift warnings. | `readOrCreateToken`, `regenerateToken`, `tokenFilePath`, `TokenReadResult`, `TokenStoreOptions` |
| `tools/` | MCP tool implementations — read-only introspection and CRUD operations | See subdirectory |
| `tools/response.ts` | Tool-response builders. `jsonContent(value)` wraps success payloads as `{ content: [{ type: 'text', text: JSON.stringify(value) }] }`. `errorContent(err, schema?, logger?)` wraps a known error as a structured `isError: true` payload whose text is `JSON.stringify(toJsonRpcError(err, ...))` so clients see the MCP error code, message, and data. `safeToolResponse(run, opts)` runs a handler body, converts known structured errors (`McpDomainError`, `McpZodError`, `ZodError`, `TaskValidationError`, `EpicNotFoundError`, `EpicCycleError`) into `errorContent`, and propagates unknown throws to outer safety nets. | `jsonContent`, `errorContent`, `safeToolResponse`, `isKnownStructuredError`, `JsonToolResponse`, `JsonToolError` |
| [`tools/meta.ts`](./meta.md) | Read-only meta tools exposing FLEET enums and configuration: `meta.repos`, `meta.taskStatuses`, `meta.dependencyConditions`. `meta.taskStatuses` and `meta.dependencyConditions` payloads are precomputed and frozen at module load; `meta.repos` reads through a caller-injected `getRepos` (wired in `index.ts` to a `createReposCache()` instance that invalidates on settings change). | `registerMetaTools`, `MetaToolsDeps` |
| `tools/tasks.ts` | Task CRUD tools (`tasks.list`, `tasks.get`, `tasks.history`, `tasks.create`, `tasks.update`, `tasks.cancel`, `tasks.requestRevision`) plus `tasks.validateSpec` for pre-flight spec checking. Dependencies are grouped into three role-specific ports — `TaskCommandPort` (create/update/cancel/onStatusTerminal), `TaskQueryPort` (list/get), `TaskHistoryPort` (getTaskChanges) — unioned into `TaskToolsDeps` for the single-argument `registerTaskTools` call. `tasks.cancel` accepts an optional `force` boolean — required when cancelling a `review` or `done` task; omitting it returns a `TaskTransitionError` (InvalidTransition) to prevent accidental loss of completed agent work. `tasks.requestRevision` is the MCP-equivalent of the in-app "Request Revision" button — only valid for tasks in `review` status; routes through `reviewOrchestration.requestRevision()` so the agent's next run receives a `<revision_feedback>` prompt block. `tasks.validateSpec` runs `SpecQualityService.validateFull()` (structural + async AI prescriptiveness check) and returns `{ valid, issues }` — skips validation and returns `{ valid: true, issues: [] }` when `spec_type` is `prompt` or `freeform`. All handlers return structured error payloads for known errors; unknown throws propagate to outer safety nets. | `registerTaskTools`, `TaskToolsDeps`, `TaskPatch`, `TaskCommandPort`, `TaskQueryPort`, `TaskHistoryPort`, `MCP_CALLER` |
| `tools/epics.ts` | Epic (task-group) CRUD tools (`epics.list`, `epics.get`, `epics.create`, `epics.update`, `epics.delete`, `epics.addTask`, `epics.removeTask`, `epics.setDependencies`). Service throws from `EpicGroupService` are rewrapped: `EpicNotFoundError` → `McpErrorCode.NotFound`, `EpicCycleError` → `McpErrorCode.Cycle`. `goal` is forwarded through unchanged — `null` reaches the service as "clear the goal," `undefined` / absent means "leave it alone." All handlers return structured error payloads for known errors. | `registerEpicTools`, `EpicToolsDeps` |
| [`transport.ts`](./transport.md) | Thin HTTP wrapper around the MCP SDK's Streamable HTTP transport — adds bearer-token auth with brute-force throttling and structured auth-failure logging (T-42), HTTP-method allow-list (POST only, T-44), explicit Origin allow-list (T-45), 2 MB request body cap (T-46), a timeout-bounded close handler (T-47), and request-context-enriched 500 error logs (T-48). `handle()` reads as a prose outline by delegating to `enforceRoute` / `checkAuthorization` / `readBoundedBody` / `buildRequestScope` / `dispatch` / `scheduleCleanup`. Accepts an optional `rateLimit: AuthRateLimit` — lazy-defaults to `createAuthRateLimit({ logger })`. | `createTransportHandler`, `TransportHandler` |
| [`index.ts`](./index.md) | `createMcpServer()` factory — wires tool registrations, binds to `127.0.0.1:<port>`, returns `{ start, stop }` handle | `createMcpServer`, `McpServerHandle`, `McpServerDeps`, `McpServerConfig` |
| `spec-guidelines.ts` | Markdown text returned by the `meta.specGuidelines` MCP tool — describes required spec sections, scope rules, and format constraints for pipeline agent specs. `## Context` is the preferred heading for the overview section; `## Overview` is also accepted for backward compatibility. | `buildSpecGuidelinesMarkdown` |
