# MCP Server

Anthropic MCP (Model Context Protocol) server for exposing BDE capabilities to Claude Code and other clients.
Source: `src/main/mcp-server/`

| Module | Purpose | Key Exports |
|--------|---------|-------------|
| [`auth.ts`](./auth.md) | Bearer-token authentication middleware with constant-time comparison for HTTP requests | `checkBearerAuth`, `AuthResult` |
| [`auth-rate-limit.ts`](./auth-rate-limit.md) | Progressive-delay rate limit for repeated 401s. Per-remote-address counter; delays kick in after 10 failures in a 60s window, escalate exponentially to a 5s cap, and clear on successful auth. Emits a one-shot `mcp.auth.brute-force-suspected` warn log at the threshold. Call sites in `transport.ts` are pending (phase 5). | `createAuthRateLimit`, `AuthRateLimit`, `computeDelayMs`, `BRUTE_FORCE_THRESHOLD`, `WINDOW_MS`, `INITIAL_DELAY_MS`, `MAX_DELAY_MS` |
| [`errors.ts`](./errors.md) | JSON-RPC error mapping for service exceptions and domain rule violations; `toJsonRpcError` enriches zod field errors with `.describe()` text when a schema is supplied (or when `parseToolArgs` wraps the failure in `McpZodError`) and logs unknown throws via an optional logger; `writeJsonRpcError` writes a full JSON-RPC 2.0 envelope to an `http.ServerResponse` | `toJsonRpcError`, `writeJsonRpcError`, `parseToolArgs`, `McpDomainError`, `McpZodError`, `McpErrorCode`, `JsonRpcErrorBody`, `JSON_RPC_*` code constants |
| [`safe-tool-handler.ts`](./safe-tool-handler.md) | Uniform error-logging wrapper for MCP tool callbacks — the MCP analog of `safeHandle()`. Logs unknown throws with the tool name before they propagate, so failures show up in `~/.bde/bde.log`. `wrapServerWithSafeToolHandlers` intercepts `McpServer.tool(...)` so no call site has to opt in. | `safeToolHandler`, `wrapServerWithSafeToolHandlers` |
| [`schemas.ts`](./schemas.md) | Zod schemas for every MCP tool argument shape; length-capped fields carry `.describe()` text for client discoverability and enriched error messages. Also exports `TASK_HISTORY_DEFAULT_LIMIT` / `TASK_HISTORY_MAX_WINDOW` used by `tools/tasks.ts` to cap `tasks.history` windows so unbounded offsets can't reach the DB. | `TaskWriteFieldsSchema`, `TaskCreateSchema`, `TaskUpdateSchema`, `EpicWriteFieldsSchema`, `TASK_HISTORY_DEFAULT_LIMIT`, `TASK_HISTORY_MAX_WINDOW`, ... |
| [`settings-events.ts`](./settings-events.md) | In-process event bus for settings-change notifications — allows main-process modules to hot-respond to config changes without IPC round-trips | `emitSettingChanged`, `onSettingChanged`, `SettingChangedEvent` |
| `test-setup.ts` | Test precondition seeding — ensures integration tests fail loudly when required repo configs are missing instead of silently skipping | `seedBdeRepo` |
| [`token-store.ts`](./token-store.md) | Persistent bearer token storage at `~/.bde/mcp-token` — generates 64-char hex token on first read, returns existing token on subsequent reads, supports regeneration | `readOrCreateToken`, `regenerateToken`, `tokenFilePath` |
| `tools/` | MCP tool implementations — read-only introspection and CRUD operations | See subdirectory |
| `tools/response.ts` | Tool-response builders. `jsonContent(value)` wraps success payloads as `{ content: [{ type: 'text', text: JSON.stringify(value) }] }`. `errorContent(err, schema?, logger?)` wraps a known error as a structured `isError: true` payload whose text is `JSON.stringify(toJsonRpcError(err, ...))` so clients see the MCP error code, message, and data. `safeToolResponse(run, opts)` runs a handler body, converts known structured errors (`McpDomainError`, `McpZodError`, `ZodError`, `TaskValidationError`, `EpicNotFoundError`, `EpicCycleError`) into `errorContent`, and propagates unknown throws to outer safety nets. | `jsonContent`, `errorContent`, `safeToolResponse`, `isKnownStructuredError`, `JsonToolResponse`, `JsonToolError` |
| [`tools/meta.ts`](./meta.md) | Read-only meta tools exposing BDE enums and configuration: `meta.repos`, `meta.taskStatuses`, `meta.dependencyConditions` | `registerMetaTools`, `defaultGetRepos`, `MetaToolsDeps` |
| `tools/tasks.ts` | Task CRUD tools (`tasks.list`, `tasks.get`, `tasks.history`, `tasks.create`, `tasks.update`, `tasks.cancel`). Dependencies are grouped into three role-specific ports — `TaskCommandPort` (create/update/cancel/onStatusTerminal), `TaskQueryPort` (list/get), `TaskHistoryPort` (getTaskChanges) — unioned into `TaskToolsDeps` for the single-argument `registerTaskTools` call. `tasks.list` forwards all filter args (`status`, `repo`, `epicId`, `tag`, `search`, `limit`, `offset`) into the data layer as a single `ListTasksOptions` object — SQL does the filtering and pagination; the tool no longer slices in memory. `tasks.history` caps `limit + offset` at `TASK_HISTORY_MAX_WINDOW` (500) and forwards both to `getTaskChanges({ limit, offset })`, which pushes pagination into SQL — oversized windows surface as `VALIDATION_FAILED` before the DB is touched. `tasks.create` accepts an optional `skipReadinessCheck` boolean for batch/admin flows that forwards as the third argument to `createTaskWithValidation`, and wraps `TaskValidationError` as `McpErrorCode.ValidationFailed` with the structural subcode (`spec-structural`, `spec-readiness`, `repo-not-configured`) carried in `data.code`. `tasks.update` takes a precise `TaskPatch` (= `z.infer<typeof TaskUpdateSchema>['patch']`) rather than an untyped record, passes `{ caller: MCP_CALLER }` through to `updateTask` so the audit trail records `changed_by='mcp'` (distinct from the IPC/UI path's `'unknown'`), auto-clears stale terminal-state fields (`completed_at`, `failure_reason`, `claimed_by`, `started_at`, `retry_count`, `fast_fail_count`, `next_eligible_at`) when a task transitions from a terminal status (`done`/`cancelled`/`failed`/`error`) to `queued` or `backlog` (parity with the `sprint:retry` IPC path), and fires `deps.onStatusTerminal(taskId, status)` when an update drives a task from a non-terminal into a terminal status — unblocking dependents, cleaning up the PR poller state, and reclaiming the worktree. `tasks.cancel` likewise passes `{ caller: MCP_CALLER }` to `cancelTask`. All handlers return structured error payloads for known errors; unknown throws propagate to outer safety nets. | `registerTaskTools`, `TaskToolsDeps`, `TaskPatch`, `TaskCommandPort`, `TaskQueryPort`, `TaskHistoryPort`, `MCP_CALLER` |
| `tools/epics.ts` | Epic (task-group) CRUD tools (`epics.list`, `epics.get`, `epics.create`, `epics.update`, `epics.delete`, `epics.addTask`, `epics.removeTask`, `epics.setDependencies`). Service throws from `EpicGroupService` are rewrapped: `EpicNotFoundError` → `McpErrorCode.NotFound`, `EpicCycleError` → `McpErrorCode.Cycle`. `goal` is forwarded through unchanged — `null` reaches the service as "clear the goal," `undefined` / absent means "leave it alone." All handlers return structured error payloads for known errors. | `registerEpicTools`, `EpicToolsDeps` |
| [`transport.ts`](./transport.md) | Thin HTTP wrapper around the MCP SDK's Streamable HTTP transport — adds bearer-token auth and structured error logging | `createTransportHandler`, `TransportHandler` |
| [`index.ts`](./index.md) | `createMcpServer()` factory — wires tool registrations, binds to `127.0.0.1:<port>`, returns `{ start, stop }` handle | `createMcpServer`, `McpServerHandle`, `McpServerDeps`, `McpServerConfig` |
