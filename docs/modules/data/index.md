# Data

Repository and query layer. All SQLite access lives here.
Source: `src/main/data/`

| Module | Purpose | Key Exports |
|--------|---------|-------------|
| [sprint-pr-ops](sprint-pr-ops.md) | PR lifecycle queries — mark done/cancelled, update mergeable state, list open PRs. `transitionTasksByPrNumber` validates each task's transition via `validateTransition` before bulk-updating; invalid-transition tasks are skipped with a warning. Audit-comparison reads use narrow projections (just the fields touched by each patch) so the PR poller's 60s loop doesn't drag `review_diff_snapshot` through every cycle. All functions accept optional `db?: Database.Database` for injection. **Write functions are async** (`markTaskDoneByPrNumber` → `Promise<string[]>`, `markTaskCancelledByPrNumber` → `Promise<string[]>`, `updateTaskMergeableState` → `Promise<void>`) — callers must `await`. **(Epic 7)** `listTasksWithOpenPrs` returns `SprintTaskPR[]`. | `markTaskDoneByPrNumber`, `markTaskCancelledByPrNumber`, `updateTaskMergeableState`, `listTasksWithOpenPrs` |
| `agent-queries.ts` | Agent run CRUD (`agent_runs`) and per-turn token tracking (`agent_run_turns`). `getAgentRunContextTokens` derives per-turn context-window size from the cumulative token columns using a SQL `LAG` window — the turn-tracker stores running totals, so context size has to be recovered as a delta. Returns both the latest-turn size (for live display) and the peak across the run (for finished agents). | `insertAgentRecord`, `getAgentMeta`, `updateAgentMeta`, `updateAgentRunCost`, `listAgents`, `listAgentRunsByTaskId`, `insertAgentRunTurn`, `listAgentRunTurns`, `getAgentRunContextTokens` |
| [sprint-task-crud](sprint-task-crud.md) | CRUD operations for sprint tasks. `listTasksRecent` projects `SPRINT_TASK_LIST_COLUMNS` so the heavy `review_diff_snapshot` blob isn't transferred on every 30s poll. `updateTask` accepts an optional `UpdateTaskOptions` argument (`{ caller? }`). `createReviewTaskFromAdhoc` stamps `promoted_to_review_at` alongside `worktree_path` / `started_at`. All exported functions accept optional `db?: Database.Database`. **Hot-path write functions are async**: `createTask`, `updateTask`, `forceUpdateTask`, `createReviewTaskFromAdhoc` → `Promise<SprintTask \| null>`. **Search safety:** `listTasks` search patterns escape LIKE metacharacters (`\`, `%`, `_`) before interpolation and use `ESCAPE '\'` on each LIKE clause, preventing pattern injection via the MCP `tasks.list` tool. | `getTask`, `listTasks`, `ListTasksOptions`, `listTasksRecent`, `createTask`, `updateTask`, `UpdateTaskOptions`, `deleteTask`, `createReviewTaskFromAdhoc` |
| `sprint-queue-ops.ts` | Queue claim/release/WIP operations. `claimTask` and `releaseTask` validate transitions via `validateTransition` before updating. **T-141 (Phase C):** async with `withRetryAsync` for SQLITE_BUSY backoff. **(Epic 7)** `claimTask` returns `SprintTaskExecution \| null`. | `claimTask`, `releaseTask`, `getActiveTaskCount`, `getQueuedTasks` |
| `sprint-query-constants.ts` | Shared column-list constants for sprint_tasks SELECT queries. `SPRINT_TASK_COLUMNS` is the full projection (includes `review_diff_snapshot`, `promoted_to_review_at`, `orphan_recovery_count`); `SPRINT_TASK_LIST_COLUMNS` excludes `review_diff_snapshot` for poll-path reads. | `SPRINT_TASK_COLUMNS`, `SPRINT_TASK_LIST_COLUMNS` |
| `sprint-agent-queries.ts` | Agent-specific health and dependency queries. All exported functions accept optional `db?: Database.Database`. **(Epic 7)** `getHealthCheckTasks` returns `SprintTaskCore[]`. **(Epic 7 perf)** `getTasksWithDependencies` accepts optional `changedTaskIds?: Set<string>`; when non-empty it issues a scoped `WHERE id IN (?)` query instead of a full table scan, enabling incremental dep-index refreshes. | `getQueueStats`, `getOrphanedTasks`, `clearStaleClaimedBy`, `clearSprintTaskFk`, `getHealthCheckTasks`, `getAllTaskIds`, `getTasksWithDependencies` |
| `sprint-planning-queries.ts` | Sprint CRUD queries. All exported functions accept optional `db?: Database.Database`. | `createSprint`, `getSprint`, `getAllSprints`, `updateSprint`, `deleteSprint`, `getSprintTasks` |
| `task-changes.ts` | Audit trail — field-level change recording. `getTaskChanges` accepts either a numeric limit (legacy) or `{ limit?, offset? }` for SQL-level `LIMIT ? OFFSET ?` pagination — upstream callers no longer fetch `limit + offset` rows and slice in memory. Functions accept optional `db?: Database.Database`. **(Epic 7)** `pruneOldChanges` uses `MS_PER_DAY`. **Perf:** `recordTaskChanges` caches the prepared INSERT statement in a `WeakMap<Database, Statement>` (keyed per db instance) so the statement is compiled once per connection rather than on every call. `recordTaskChangesBulk` already prepared once per invocation; both paths now pay zero prepare overhead on the hot path. | `recordTaskChanges`, `recordTaskChangesBulk`, `getTaskChanges`, `GetTaskChangesOptions`, `pruneOldChanges` |
| `task-group-queries.ts` | Task group (epic) CRUD and dependency management. Functions accept optional `db?: Database.Database`. `CreateGroupInput.goal` / `UpdateGroupInput.goal` are typed `string \| null \| undefined`: `null` writes SQL NULL (create) or clears the column (update); `undefined` on update is filtered out of the SET clause so callers can omit fields without accidentally wiping existing values. Row hydration routes `depends_on` through `sanitizeEpicDependsOn` (from `src/shared/`) so malformed JSON or off-schema entries degrade gracefully instead of being cast blindly to `EpicDependency[]`. **EP-9:** `getGroupTasks` projects `SPRINT_TASK_LIST_COLUMNS` (no `SELECT *`) so epic detail loads don't drag the `review_diff_snapshot` blob. | `createGroup`, `listGroups`, `getGroup`, `updateGroup`, `deleteGroup`, `addTaskToGroup`, `removeTaskFromGroup`, `getGroupTasks`, `queueAllGroupTasks`, `reorderGroupTasks`, `addGroupDependency`, `removeGroupDependency`, `updateGroupDependencyCondition`, `getGroupsWithDependencies` |
| `webhook-queries.ts` | Webhook CRUD. All exported functions accept optional `db?: Database.Database`. Row hydration routes the `events` TEXT column through `parseWebhookEvents` (shape validator) and the `secret` column through `decryptWebhookSecret` (safeStorage-backed decryption that transparently falls through to plaintext for legacy rows). Writes encrypt the HMAC secret via `encryptWebhookSecret` (safeStorage + ENC: prefix); throws `EncryptionUnavailableError` when `safeStorage` is unavailable — cleartext fallback was removed as a security regression. | `listWebhooks`, `createWebhook`, `updateWebhook`, `deleteWebhook`, `getWebhookById`, `getWebhooks`, `parseWebhookEvents`, `EncryptionUnavailableError` |
| `dashboard-queries.ts` | Dashboard analytics queries — hourly completions and recent agent events. Transforms raw DB rows to camelCase `DashboardEvent` (including JSON-parsed `payload`) before returning. Functions accept optional `db?: Database.Database`. | `getCompletionsPerHour`, `getRecentEvents` |
| `event-queries.ts` | Agent event persistence — append, batch insert, query with filtering/pagination, prune by age or agent IDs. `insertEventBatch` caches the prepared INSERT statement per-db via a `WeakMap<Database, CachedStatements>` so the same SQL is compiled only once per db instance; test DBs are freed automatically when the WeakMap entry is GC'd. All functions take `db: Database.Database` as first parameter for testability. | `appendEvent`, `insertEventBatch`, `getEventHistory`, `queryEvents`, `pruneOldEvents`, `pruneEventsByAgentIds`, `EventBatchItem`, `EventRow`, `QueryEventsOptions`, `QueryEventsResult`, `DEFAULT_EVENT_HISTORY_LIMIT` |
| `reporting-queries.ts` | Analytics queries for sprint tasks. All functions accept optional `db?: Database.Database`. | `getDoneTodayCount`, `getFailureReasonBreakdown`, `getTaskRuntimeStats`, `getSuccessRateBySpecType`, `getDailySuccessRate` |
| `sprint-maintenance.ts` | SQLite maintenance ops for sprint_tasks: diff-snapshot pruning, test-artifact cleanup. **(Epic 7)** Uses `MS_PER_DAY` from `shared/time.ts`. | `pruneOldDiffSnapshots`, `cleanTestArtifacts`, `DIFF_SNAPSHOT_RETENTION_DAYS` |
| `sprint-maintenance-facade.ts` | Stable re-export path for maintenance utilities (snapshot pruning, FK cleanup, update allowlist, test cleanup) | `cleanTestArtifacts`, `pruneOldDiffSnapshots`, `DIFF_SNAPSHOT_RETENTION_DAYS`, `UPDATE_ALLOWLIST`, `clearSprintTaskFk` |
| `sprint-task-mapper.ts` | Row→SprintTask mapper. `mapRowToTask` runtime-validates the critical domain fields (`id`, `status`, `priority`, `repo`, `title`) and throws `Invalid sprint_tasks row: …` on corruption so downstream exhaustiveness checks can trust the shape. `mapRowsToTasks` drops invalid rows and logs each via the injected sprint-queries logger — one corrupted row cannot break a list query. `serializeFieldForStorage` is unchanged. | `mapRowToTask`, `mapRowsToTasks`, `serializeFieldForStorage` |
| `sqlite-retry.ts` | SQLite retry wrappers with exponential backoff for SQLITE_BUSY errors. `withRetry` (sync, blocks via `Atomics.wait` — cold-path only) and `withRetryAsync` (async, yields via `setTimeout` — use on hot paths). `RetryOptions` accepts an optional `logger?: { warn: (msg: string) => void }` so callers can surface per-retry-round contention in `~/.fleet/fleet.log` without coupling the module to `src/main/logger.ts`. The minimal interface is a structural subtype of `Logger`, `console`, and test mocks. | `withRetry`, `withRetryAsync`, `RetryOptions` |
| `data-utils.ts` | Shared error-handling utility for the data layer. Eliminates the repeated try/catch log-and-return pattern. | `withDataLayerError` |
| `migrations/loader.ts` | Loads all migration files via `import.meta.glob` (bundler-safe) and validates the full sequence is contiguous 1..N. Also exports `getPendingMigrations(migrations, currentVersion)` which filters to versions above `currentVersion`, sorts, and throws on any version gap before migrations run. | `loadMigrations`, `getPendingMigrations`, `Migration` |
| `unit-of-work.ts` | Transaction abstraction. `IUnitOfWork.runInTransaction(work)` wraps `work` in a `better-sqlite3` transaction so callers (terminal-handler, auto-merge-coordinator) that hold an injected repository can group multiple writes atomically without importing `getDb` directly. Default factory `createUnitOfWork()` lazily resolves `getDb()` per call. T-17 (Phase 1b). | `IUnitOfWork`, `createUnitOfWork` |
| `review-repository.ts` | Cache for structured AI review results. `createReviewRepository(db)` returns an `IReviewRepository` that stores and retrieves `ReviewResult` keyed by `(taskId, commitSha)`. `getCached` validates the parsed `findings_json` shape (must be a `ReviewFindings` object with a `perFile` array) and returns `{ perFile: [] }` on unexpected shape rather than propagating a broken object across IPC. | `createReviewRepository`, `IReviewRepository` |
| `sprint-task-repository.ts` | Repository abstraction layer — `IAgentTaskRepository`, `ISprintPollerRepository`, `IDashboardRepository`, and the composite `ISprintTaskRepository`. Factory `createSprintTaskRepository()` delegates to sprint-queries. **T-133:** removed singleton helpers (`getSharedSprintTaskRepository`, `setSharedSprintTaskRepository`, `_resetSharedSprintTaskRepository`) — consumers receive the instance from the composition root, not from a lazy module-level singleton. `claimTask` accepts optional `maxActive` for atomic DB-level WIP enforcement. `IAgentTaskRepository` also exposes `getQueueStats()` so the drain loop can report queue depth in drain-paused broadcasts. `IDashboardRepository.listTasks` accepts either a bare status string (legacy) or a `ListTasksOptions` object. `IAgentTaskRepository.updateTask` accepts an optional third `UpdateTaskOptions` argument — `{ caller? }`. **All hot-path write method signatures are async**: `updateTask`/`claimTask` → `Promise<SprintTask | null>`; `createTask`/`releaseTask`/`forceUpdateTask`/`createReviewTaskFromAdhoc` → `Promise<SprintTask | null>`; `markTaskDoneByPrNumber`/`markTaskCancelledByPrNumber` → `Promise<string[]>`; `updateTaskMergeableState` → `Promise<void>`. **(Epic 7)** `IAgentTaskRepository.claimTask` returns `SprintTaskExecution \| null`; `IDashboardRepository.getHealthCheckTasks` returns `SprintTaskCore[]`; `ISprintPollerRepository.listTasksWithOpenPrs` returns `SprintTaskPR[]`. **(Epic 7 perf)** `IAgentTaskRepository.getTasksWithDependencies` accepts optional `changedTaskIds?: Set<string>` — forwards to `sprint-agent-queries` for incremental dep-index refreshes. | `ISprintTaskRepository`, `IAgentTaskRepository`, `ISprintPollerRepository`, `IDashboardRepository`, `createSprintTaskRepository`, `ListTasksOptions`, `UpdateTaskOptions` |
