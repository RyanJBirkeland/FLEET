# Data

Repository and query layer. All SQLite access lives here.
Source: `src/main/data/`

| Module | Purpose | Key Exports |
|--------|---------|-------------|
| [sprint-pr-ops](sprint-pr-ops.md) | PR lifecycle queries — mark done/cancelled, update mergeable state, list open PRs. `transitionTasksByPrNumber` validates each task's transition via `validateTransition` before bulk-updating; invalid-transition tasks are skipped with a warning. Audit-comparison reads use narrow projections (just the fields touched by each patch) so the PR poller's 60s loop doesn't drag `review_diff_snapshot` through every cycle. All functions accept optional `db?: Database.Database` for injection. **Write functions are async** (`markTaskDoneByPrNumber` → `Promise<string[]>`, `markTaskCancelledByPrNumber` → `Promise<string[]>`, `updateTaskMergeableState` → `Promise<void>`) — callers must `await`. **(Epic 7)** `listTasksWithOpenPrs` returns `SprintTaskPR[]`. | `markTaskDoneByPrNumber`, `markTaskCancelledByPrNumber`, `updateTaskMergeableState`, `listTasksWithOpenPrs` |
| `agent-queries.ts` | Agent run CRUD (`agent_runs`) and per-turn token tracking (`agent_run_turns`). `getAgentRunContextTokens` derives per-turn context-window size from the cumulative token columns using a SQL `LAG` window — the turn-tracker stores running totals, so context size has to be recovered as a delta. Returns both the latest-turn size (for live display) and the peak across the run (for finished agents). | `insertAgentRecord`, `getAgentMeta`, `updateAgentMeta`, `updateAgentRunCost`, `listAgents`, `listAgentRunsByTaskId`, `insertAgentRunTurn`, `listAgentRunTurns`, `getAgentRunContextTokens` |
| [sprint-task-crud](sprint-task-crud.md) | CRUD operations for sprint tasks. `listTasks` takes either a bare status string (legacy) or a `ListTasksOptions` object — `{ status?, repo?, epicId?, tag?, search?, limit?, offset? }` — and pushes every filter and the pagination window into SQL (tag match uses `json_each`; search is case-insensitive `LIKE` on title + spec). `listTasksRecent` (the renderer poll path) projects `SPRINT_TASK_LIST_COLUMNS` so the heavy `review_diff_snapshot` blob isn't transferred on every 30s poll. `updateTask` accepts an optional `UpdateTaskOptions` argument (`{ caller? }`); `caller` lands in the `task_changes` audit trail as `changed_by` so MCP-originated writes (`'mcp'`) surface distinctly from IPC/UI ones instead of collapsing to `'unknown'`. All exported functions accept optional `db?: Database.Database`. **Hot-path write functions are async**: `createTask` → `Promise<SprintTask | null>`, `updateTask` → `Promise<SprintTask | null>`, `forceUpdateTask` → `Promise<SprintTask | null>`, `createReviewTaskFromAdhoc` → `Promise<SprintTask | null>`. | `getTask`, `listTasks`, `ListTasksOptions`, `listTasksRecent`, `createTask`, `updateTask`, `UpdateTaskOptions`, `deleteTask`, `createReviewTaskFromAdhoc` |
| `sprint-queue-ops.ts` | Queue claim/release/WIP operations. `claimTask` and `releaseTask` validate transitions via `validateTransition` before updating — invalid transitions log a warning and return null instead of silently no-oping. All exported functions accept optional `db?: Database.Database`. **T-141 (Phase C):** `claimTask` and `releaseTask` are async and use `withRetryAsync` for SQLITE_BUSY backoff — retry delays use `setTimeout` instead of blocking `Atomics.wait` so the Electron main thread stays responsive under contention. **(Epic 7)** `claimTask` returns `SprintTaskExecution \| null` — drain-loop callers that read execution fields after claiming get the narrower, intent-revealing type. | `claimTask`, `releaseTask`, `getActiveTaskCount`, `getQueuedTasks` |
| `sprint-query-constants.ts` | Shared column-list constants for sprint_tasks SELECT queries. `SPRINT_TASK_COLUMNS` is the full projection (includes `review_diff_snapshot` and `orphan_recovery_count`); `SPRINT_TASK_LIST_COLUMNS` excludes the heavy `review_diff_snapshot` blob for poll-path reads. **EP-9:** file header carries an explicit "no `SELECT *` against `sprint_tasks`" warning — list paths must use the LIST variant so renderer polls don't transfer the snapshot blob. | `SPRINT_TASK_COLUMNS`, `SPRINT_TASK_LIST_COLUMNS` |
| `sprint-agent-queries.ts` | Agent-specific health and dependency queries. All exported functions accept optional `db?: Database.Database`. **(Epic 7)** `getHealthCheckTasks` returns `SprintTaskCore[]`; uses `MS_PER_HOUR` from `shared/time.ts` for the one-hour lookback. | `getQueueStats`, `getOrphanedTasks`, `clearStaleClaimedBy`, `clearSprintTaskFk`, `getHealthCheckTasks`, `getAllTaskIds`, `getTasksWithDependencies` |
| `sprint-planning-queries.ts` | Sprint CRUD queries. All exported functions accept optional `db?: Database.Database`. | `createSprint`, `getSprint`, `getAllSprints`, `updateSprint`, `deleteSprint`, `getSprintTasks` |
| `task-changes.ts` | Audit trail — field-level change recording. `getTaskChanges` accepts either a numeric limit (legacy) or `{ limit?, offset? }` for SQL-level `LIMIT ? OFFSET ?` pagination — upstream callers no longer fetch `limit + offset` rows and slice in memory. Functions accept optional `db?: Database.Database`. **(Epic 7)** `pruneOldChanges` uses `MS_PER_DAY`. | `recordTaskChanges`, `recordTaskChangesBulk`, `getTaskChanges`, `GetTaskChangesOptions`, `pruneOldChanges` |
| `task-group-queries.ts` | Task group (epic) CRUD and dependency management. Functions accept optional `db?: Database.Database`. `CreateGroupInput.goal` / `UpdateGroupInput.goal` are typed `string \| null \| undefined`: `null` writes SQL NULL (create) or clears the column (update); `undefined` on update is filtered out of the SET clause so callers can omit fields without accidentally wiping existing values. Row hydration routes `depends_on` through `sanitizeEpicDependsOn` (from `src/shared/`) so malformed JSON or off-schema entries degrade gracefully instead of being cast blindly to `EpicDependency[]`. **EP-9:** `getGroupTasks` projects `SPRINT_TASK_LIST_COLUMNS` (no `SELECT *`) so epic detail loads don't drag the `review_diff_snapshot` blob. | `createGroup`, `listGroups`, `getGroup`, `updateGroup`, `deleteGroup`, `addTaskToGroup`, `removeTaskFromGroup`, `getGroupTasks`, `queueAllGroupTasks`, `reorderGroupTasks`, `addGroupDependency`, `removeGroupDependency`, `updateGroupDependencyCondition`, `getGroupsWithDependencies` |
| `webhook-queries.ts` | Webhook CRUD. All exported functions accept optional `db?: Database.Database`. Row hydration routes the `events` TEXT column through `parseWebhookEvents` (shape validator) and the `secret` column through `decryptWebhookSecret` (safeStorage-backed decryption that transparently falls through to plaintext for legacy rows). Writes encrypt the HMAC secret via `encryptWebhookSecret` (safeStorage + ENC: prefix) when platform encryption is available, falling back to plaintext with a warn log when it is not. | `listWebhooks`, `createWebhook`, `updateWebhook`, `deleteWebhook`, `getWebhookById`, `getWebhooks`, `parseWebhookEvents` |
| `dashboard-queries.ts` | Dashboard analytics queries — hourly completions and recent agent events. Transforms raw DB rows to camelCase `DashboardEvent` (including JSON-parsed `payload`) before returning. Functions accept optional `db?: Database.Database`. | `getCompletionsPerHour`, `getRecentEvents` |
| `reporting-queries.ts` | Analytics queries for sprint tasks. All functions accept optional `db?: Database.Database`. | `getDoneTodayCount`, `getFailureReasonBreakdown`, `getTaskRuntimeStats`, `getSuccessRateBySpecType`, `getDailySuccessRate` |
| `sprint-maintenance.ts` | SQLite maintenance ops for sprint_tasks: diff-snapshot pruning, test-artifact cleanup. **(Epic 7)** Uses `MS_PER_DAY` from `shared/time.ts`. | `pruneOldDiffSnapshots`, `cleanTestArtifacts`, `DIFF_SNAPSHOT_RETENTION_DAYS` |
| `sprint-maintenance-facade.ts` | Stable re-export path for maintenance utilities (snapshot pruning, FK cleanup, update allowlist, test cleanup) | `cleanTestArtifacts`, `pruneOldDiffSnapshots`, `DIFF_SNAPSHOT_RETENTION_DAYS`, `UPDATE_ALLOWLIST`, `clearSprintTaskFk` |
| `sprint-task-mapper.ts` | Row→SprintTask mapper. `mapRowToTask` runtime-validates the critical domain fields (`id`, `status`, `priority`, `repo`, `title`) and throws `Invalid sprint_tasks row: …` on corruption so downstream exhaustiveness checks can trust the shape. `mapRowsToTasks` drops invalid rows and logs each via the injected sprint-queries logger — one corrupted row cannot break a list query. `serializeFieldForStorage` is unchanged. | `mapRowToTask`, `mapRowsToTasks`, `serializeFieldForStorage` |
| `sqlite-retry.ts` | SQLite retry wrappers with exponential backoff for SQLITE_BUSY errors. `withRetry` (sync, blocks via `Atomics.wait` — cold-path only) and `withRetryAsync` (async, yields via `setTimeout` — use on hot paths). `RetryOptions` accepts an optional `logger?: { warn: (msg: string) => void }` so callers can surface per-retry-round contention in `~/.bde/bde.log` without coupling the module to `src/main/logger.ts`. The minimal interface is a structural subtype of `Logger`, `console`, and test mocks. | `withRetry`, `withRetryAsync`, `RetryOptions` |
| `data-utils.ts` | Shared error-handling utility for the data layer. Eliminates the repeated try/catch log-and-return pattern. | `withDataLayerError` |
| `migrations/loader.ts` | Loads all migration files via `import.meta.glob` (bundler-safe) and validates the full sequence is contiguous 1..N. Also exports `getPendingMigrations(migrations, currentVersion)` which filters to versions above `currentVersion`, sorts, and throws on any version gap before migrations run. | `loadMigrations`, `getPendingMigrations`, `Migration` |
| `unit-of-work.ts` | Transaction abstraction. `IUnitOfWork.runInTransaction(work)` wraps `work` in a `better-sqlite3` transaction so callers (terminal-handler, auto-merge-coordinator) that hold an injected repository can group multiple writes atomically without importing `getDb` directly. Default factory `createUnitOfWork()` lazily resolves `getDb()` per call. T-17 (Phase 1b). | `IUnitOfWork`, `createUnitOfWork` |
| `review-repository.ts` | Cache for structured AI review results. `createReviewRepository(db)` returns an `IReviewRepository` that stores and retrieves `ReviewResult` keyed by `(taskId, commitSha)`. `getCached` validates the parsed `findings_json` shape (must be a `ReviewFindings` object with a `perFile` array) and returns `{ perFile: [] }` on unexpected shape rather than propagating a broken object across IPC. | `createReviewRepository`, `IReviewRepository` |
| `sprint-task-repository.ts` | Repository abstraction layer — `IAgentTaskRepository`, `ISprintPollerRepository`, `IDashboardRepository`, and the composite `ISprintTaskRepository`. Factory `createSprintTaskRepository()` delegates to sprint-queries. **T-133:** removed singleton helpers (`getSharedSprintTaskRepository`, `setSharedSprintTaskRepository`, `_resetSharedSprintTaskRepository`) — consumers receive the instance from the composition root, not from a lazy module-level singleton. `claimTask` accepts optional `maxActive` for atomic DB-level WIP enforcement. `IAgentTaskRepository` also exposes `getQueueStats()` so the drain loop can report queue depth in drain-paused broadcasts. `IDashboardRepository.listTasks` accepts either a bare status string (legacy) or a `ListTasksOptions` object. `IAgentTaskRepository.updateTask` accepts an optional third `UpdateTaskOptions` argument — `{ caller? }`. **All hot-path write method signatures are async**: `updateTask`/`claimTask` → `Promise<SprintTask | null>`; `createTask`/`releaseTask`/`forceUpdateTask`/`createReviewTaskFromAdhoc` → `Promise<SprintTask | null>`; `markTaskDoneByPrNumber`/`markTaskCancelledByPrNumber` → `Promise<string[]>`; `updateTaskMergeableState` → `Promise<void>`. **(Epic 7)** `IAgentTaskRepository.claimTask` returns `SprintTaskExecution \| null`; `IDashboardRepository.getHealthCheckTasks` returns `SprintTaskCore[]`; `ISprintPollerRepository.listTasksWithOpenPrs` returns `SprintTaskPR[]`. | `ISprintTaskRepository`, `IAgentTaskRepository`, `ISprintPollerRepository`, `IDashboardRepository`, `createSprintTaskRepository`, `ListTasksOptions`, `UpdateTaskOptions` |
