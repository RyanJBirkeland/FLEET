# Handlers

IPC handler modules. Thin wrappers — receive IPC calls, delegate to services, return results.
Source: `src/main/handlers/`

| Module | Purpose | Key Exports |
|--------|---------|-------------|
| `agent-manager-handlers.ts` | Agent manager orchestration IPC handlers — status, kill, metrics, reloadConfig, checkpoint, and triggerDrain. `agent-manager:triggerDrain` immediately runs one drain tick, used by the "Check now" pipeline header button. All handlers are no-ops when the agent manager is not available. | `registerAgentManagerHandlers` |
| `git-handlers.ts` | Git and GitHub IPC handlers — source control, PR polling, GitHub API proxy. Includes `git:checkInstalled` which runs `git --version` to verify the CLI is present (returns `true`/`false`). **UX:** adds `git:diffBetweenRefs` — runs `git diff <fromRef>..<toRef>` in a validated worktree path; used by Code Review "Since last review" toggle. Path validated via `validateWorktreePath`. **Phase 6.5:** adds `git:fileLog` — runs `git log --follow -n <max20>` for the IDE Insight Rail's RECENT COMMITS section; validates that `cwd` is an absolute non-traversal path and that `filePath` resolves within `cwd`, returns up to `n` commits as `{ hash, shortHash, subject, author, date }[]`. | `registerGitHandlers`, `GitHandlersDeps` |
| `window-handlers.ts` | Window and playground IPC handlers — external URL gating (`window:openExternal`), worktree directory opener (`window:openPath` — validates via `validateWorktreePath` before calling `shell.openPath`), window title, open-in-browser for playground HTML, re-sanitize user-edited HTML. | `registerWindowHandlers` |
| `synthesizer-handlers.ts` | IPC handlers for AI spec generation and revision (`synthesizer:generate`, `synthesizer:revise`, `synthesizer:cancel`). Validates request payloads before delegating to `spec-synthesizer` service. | `registerSynthesizerHandlers` |
| `agent-handlers.ts` | Agent lifecycle IPC handlers — spawn adhoc agents, steer/kill agents, list/read agent history, promote to code review. `local:spawnClaudeAgent` validates `repoPath` against the configured-repo allowlist via `validateRepoPath` before forwarding to `spawnAdhocAgent` — out-of-allowlist paths return an IPC error without spawning. `agent:steer` enforces image payload limits: max 5 images per request, max 5 MB each (base64), and MIME type restricted to `image/png`, `image/jpeg`, `image/gif`, `image/webp`. `agent:history` calls `flushAgentEventBatcher()` before reading so a history fetch issued immediately after spawn can't race the 100 ms SQLite-batch timer, then shape-guards every parsed `agent_events.payload` row against the `AgentEvent` union and drops malformed entries with a warn log. `agents:import` uses `parseArgs` to validate `{ meta: object, content: string }` at the wire boundary. `agents:testLocalEndpoint` validates the URL hostname against a loopback allowlist (`localhost`, `127.0.0.1`, `::1`) — `0.0.0.0` is explicitly excluded as it is not a loopback address. `repo` parameter is required (no fallback constructor call). | `registerAgentHandlers`, `PromoteToReviewResult`, `testLocalEndpoint` |
| `review.ts` | Thin IPC adapters for the Code Review Station — diff/commits/file queries, freshness/auto-review checks, and action handlers (`review:mergeLocally`, `review:createPr`, `review:requestRevision`, `review:discard`, `review:shipIt`, `review:shipBatch`, `review:rebase`, `review:markShippedOutsideFleet`). `review:shipBatch` validates every task id and delegates to `services/review-ship-batch.ts`. `review:markShippedOutsideFleet` transitions a `review`-status task to `done` without any git operation — used when work was shipped outside of FLEET. The diff/commits/file-diff handlers call `assertWorktreeExists` after path validation so a missing worktree directory surfaces as `WorktreeMissingError` instead of the misleading `spawn git ENOENT`. `review:getDiff`, `review:getCommits`, and `review:getFileDiff` resolve the default branch from the worktree path via `resolveDefaultBranch` — the renderer no longer sends a `base` parameter, so repos using `master` or any non-`main` default branch work correctly. `review:getDiff` and `review:getCommits` use `parseReviewWorktreeArgs` (exported) to validate `worktreePath`. `review:getFileDiff` uses `parseReviewFileDiffArgs` (exported) to validate `worktreePath` and `filePath` (via `validateFilePath`). `buildAgentEnv()` is called inside each handler closure at request time rather than once at handler registration — every request captures a fresh env snapshot. | `registerReviewHandlers`, `ReviewHandlersDeps`, `parseReviewWorktreeArgs`, `parseReviewFileDiffArgs` |
| `review-assistant.ts` | Chat stream IPC handler for the interactive code review assistant. Resolves the reviewer model per-call via injected `resolveAgentRuntime` so Settings → Models → Reviewer changes apply without a restart. Streams with `tools: ['Read', 'Grep', 'Glob']` and explicitly opts into `permissionMode: 'bypassPermissions'` / `allowDangerouslySkipPermissions: true` — the read-only tool list makes bypass safe and keeps the chat headless. Also passes `settingSources: ['user', 'local']` (inherits user-scoped MCP servers, hooks, and permissions from `~/.claude/settings.json`; `'project'` excluded so repo CLAUDE.md isn't double-injected) and `maxBudgetUsd: 2.0` (hard spend ceiling for user-triggered chat sessions) into the SDK. | `registerReviewAssistantHandler`, `ChatStreamDeps`, `buildChatStreamDeps` |
| `sprint-ipc-schemas.ts` | Zod schemas for IPC wire-boundary validation. `CreateTaskInputSchema` (re-exports `TaskWriteFieldsSchema`), `WorkflowStepSchema`, `WorkflowTemplateSchema`, `BatchImportTaskSchema`. Import from here rather than defining inline validators in handler files. | `CreateTaskInputSchema`, `WorkflowStepSchema`, `WorkflowTemplateSchema`, `BatchImportTaskSchema`, `TaskDependencySchema` |
| `sprint-local.ts` | Sprint task lifecycle IPC handlers — create, update, queue, log reads, dependency management. `sprint:create` and `sprint:createWorkflow` both delegate to Zod schemas in `sprint-ipc-schemas.ts` for full shape validation (T-26). `sprint:create` passes `taskStateService` through to `createTaskWithValidation` for future use. `sprint:update` uses `parseSprintUpdateArgs` (plain-object + string check). Operator escape-hatches `sprint:forceFailTask` and `sprint:forceDoneTask` bypass the state-machine transition check via `forceUpdateTask`, still routing through `onStatusTerminal` so dependents unblock. **(Epic 6)** `sprint:claimTask` and `sprint:forceReleaseClaim` are now thin wrappers — business logic extracted to `buildClaimedTask` and `forceReleaseClaim` in `sprint-service.ts`. `repo` parameter is required (no fallback constructor call). **UX:** adds `tasks:getLastPrompt` handler — returns `{ prompt: string | null }` for a task's most recently rendered agent prompt (read from `sprint_tasks.last_rendered_prompt`). | `registerSprintLocalHandlers`, `parseCreateWorkflowArgs` |
| `sprint-batch-handlers.ts` | Batch sprint task operations — `sprint:batchUpdate` (multi-op update/delete), `sprint:batchImport` (bulk task creation). `sprint:batchImport` uses `BatchImportTaskSchema` from `sprint-ipc-schemas.ts` to fully validate each element including `depType`, `playgroundEnabled`, `priority`, and `tags` (T-25). | `registerSprintBatchHandlers`, `BatchHandlersDeps` |
| `sprint-retry-handler.ts` | Handles `sprint:retry` — thin wrapper: validates task ID then delegates to `retryTask()` in `sprint-service`. **(Epic 6)** All retry logic (status guard, git worktree prune, branch cleanup, state reset) extracted to `sprint-service.retryTask`. | `registerSprintRetryHandler` |
| `sprint-spec.ts` | Sprint spec I/O and path validation — validates relative spec file paths for containment and existence. Error messages use repo-agnostic language. | `validateSpecPath`, `GeneratePromptRequest`, `GeneratePromptResponse` |
| `sprint-export-handlers.ts` | CSV export IPC handler (`sprint:exportTaskHistory`) — fetches a task's change history via `getTaskChanges` and writes a timestamped CSV through the injected dialog service. | `registerSprintExportHandlers`, `ExportHandlersDeps` |
| `planner-import.ts` | Markdown plan import — parses H1/H2/H3 plan files into epics + tasks, validates repo against configured list | `registerPlannerImportHandlers`, `parsePlanMarkdown`, `importPlanFile` |
| `auth-handlers.ts` | Auth and onboarding prerequisite IPC handlers — `auth:status` check (Claude CLI + token), `onboarding:checkGhCli` (gh CLI availability, version, and authentication status via `gh auth status`) | `registerAuthHandlers`, `registerOnboardingHandlers` |
| `config-handlers.ts` | Settings CRUD and profiles IPC handlers. `settings:get` blocks sensitive keys (returns `null`). `settings:hasSecret` checks presence of sensitive keys without exposing plaintext. `settings:getEncryptionStatus` returns safeStorage encryption availability on the system. Path-validates `agentManager.worktreeBase`. Emits local `setting-changed` events via `settings-events.ts` on successful writes (`settings:set`, `settings:setJson`, `settings:delete`) to enable hot-toggle in main-process modules. Also registers `mcp:getToken`, `mcp:regenerateToken`, and `mcp:getTokenRotatedAt` — delegates to `token-store`; regenerate cycles `mcp.enabled` off/on to restart the server if it is running; `mcp:getTokenRotatedAt` returns the ISO timestamp written by the last `regenerateToken` call (or `null`). `settings:setJson` uses `parseSetJsonArgs` (exported) to block writes to sensitive keys and reject serialised values exceeding 1 MB. | `registerConfigHandlers`, `parseSetJsonArgs` |
| `group-handlers.ts` | Thin IPC wrapper for epic/task-group operations. Holds the module-level `EpicGroupService` singleton and delegates all channel logic to it. Exposes `getEpicGroupService()` for MCP access to the shared service instance. | `registerGroupHandlers`, `getEpicGroupService` |
| `workbench.ts` | Task Workbench IPC handlers — AI-assisted task creation (`workbench:chatStream`, `workbench:cancelStream`, `workbench:generateSpec`, `workbench:checkSpec`, `workbench:checkOperational`, `workbench:researchRepo`, `workbench:extractPlan`). `workbench:chatStream` resolves the copilot model per-call via `resolveAgentRuntime('copilot')` and passes it to `getCopilotSdkOptions`, so Settings → Models → Copilot changes apply without a restart. `workbench:checkSpec` uses `parseArgs` to validate `{ title, repo, spec }` strings at the wire boundary. `workbench:researchRepo` uses `parseResearchRepoArgs` (exported) to validate non-empty `query` and `repo`. `workbench:chatStream` uses `parseChatStreamArgs` (exported) to validate `messages` array and `formContext.repo`. `workbench:checkOperational` uses `parseCheckOperationalArgs` to validate non-empty `repo`. `workbench:generateSpec` uses `parseGenerateSpecArgs` to validate non-empty `title` and `repo`. `workbench:extractPlan` enforces a 200 KB markdown size limit. The `activeStreams` Map is now scoped to the handler registration closure (not module-global). `mapQualityResult` delegates to `classifyIssues` helper to eliminate duplicated filter/classify logic. | `registerWorkbenchHandlers`, `parseResearchRepoArgs`, `parseChatStreamArgs` |
| `terminal-handlers.ts` | Terminal PTY IPC handlers — `terminal:create` spawns a PTY with validated shell and optional `cwd`, `terminal:write` forwards stdin, `terminal:resize` updates dimensions, `terminal:kill` tears down. `terminal:create` uses `parseTerminalCreateArgs` and `validateTerminalCwd` (exported) to reject `cwd` values outside configured repo localPaths, pipeline worktree base, or adhoc worktree base. | `registerTerminalHandlers`, `validateTerminalCwd` |
| `repo-discovery.ts` | Repository discovery IPC handlers — `repos:scanLocal` (scan directories for git repos), `repos:listGithub` (enumerate GitHub repos via `gh repo list`), `repos:clone` (clone with progress broadcast). `listGithubRepos` validates each JSON entry against a handwritten `isGhRepoEntry` type guard before use — invalid entries (non-array output, missing/null `owner`, non-string `owner.login`) are dropped with a warn log instead of throwing. | `registerRepoDiscoveryHandlers`, `scanLocalRepos`, `listGithubRepos`, `cloneRepo` |
| `ide-fs-handlers.ts` | IDE filesystem IPC handlers (`fs:watchDir`, `fs:readFile`, `fs:writeFile`, etc.). `validateIdeRoot` default-denies; an IDE root is accepted only if it matches a configured repo `localPath` or has been approved this session via `rememberApprovedIdeRoot` (which `fs:openDirectoryDialog` invokes on user pick). Exports `validateIdePath` for caller-side containment checks. | `registerIdeFsHandlers`, `getIdeRootPath`, `validateIdePath`, `validateIdeRoot`, `rememberApprovedIdeRoot` |
| `claude-config-handlers.ts` | Claude Code settings bridge (`claude:getConfig`, `claude:setPermissions`). `setPermissions` validates every rule against a known-prefix allowlist (`Bash`, `Read`, `Write`, `Edit`, `MultiEdit`, `NotebookEdit`, `Glob`, `Grep`, `WebFetch`, `WebSearch` — optionally followed by a parenthesised match spec) and caps rule length at 500 characters before writing to `~/.claude/settings.json`. | `registerClaudeConfigHandlers` |
| `webhook-handlers.ts` | Webhook management IPC handlers (`webhook:list`, `webhook:create`, `webhook:update`, `webhook:delete`, `webhook:test`). `validateWebhookUrl` guards against SSRF — rejects non-http(s) schemes, loopback names (`localhost`, `0.0.0.0`), IPv4 private/link-local ranges (using `net.isIPv4` to avoid octal/decimal bypass vectors), and IPv6 private addresses (loopback `::1`, IPv4-mapped `::ffff:`, ULA `fc`/`fd`, link-local `fe80`) via `net.isIPv6`. | `registerWebhookHandlers` |
| `update-handlers.ts` | Auto-update IPC handlers — `updates:checkForUpdates` triggers a manual update check, `updates:install` calls `quitAndInstall` to apply a downloaded update. Both delegate to `UpdaterService`. | `registerUpdateHandlers` |
| `pr-groups.ts` | PR Group CRUD and build IPC handlers — `prGroups:list`, `prGroups:create`, `prGroups:update`, `prGroups:addTask`, `prGroups:removeTask`, `prGroups:build`, `prGroups:delete`. All mutations delegate to `pr-group-queries`; `prGroups:build` delegates to `PrGroupBuildService`. | `registerPrGroupHandlers`, `PrGroupHandlersDeps` |
