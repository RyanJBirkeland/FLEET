# Lib — Main

Utility functions and shared helpers for the main process.
Source: `src/main/lib/` (and the shared SDK streaming helper at `src/main/sdk-streaming.ts`).

| Module | Purpose | Key Exports |
|--------|---------|-------------|
| `../sdk-streaming.ts` | Shared Claude Agent SDK streaming helpers. `SdkStreamingOptions.model` is **required** — every call site must resolve it from `agents.backendConfig` via `resolveAgentRuntime(type).model`. No silent default. `permissionMode` and `allowDangerouslySkipPermissions` are per-call opt-ins (undefined → SDK default, which prompts) — the previously hard-coded `bypassPermissions` was removed so every caller relying on headless tool use must explicitly opt in beside a vetted narrow `tools` list. | `runSdkStreaming`, `runSdkOnce`, `SdkStreamingOptions`, `ToolUseEvent` |
| `env-utils.ts` | Environment setup — PATH augmentation (with optional `BDE_EXTRA_PATHS` prefix), OAuth token management with symlink/size guards, agent environment building. `refreshOAuthTokenFromKeychain` tracks consecutive Keychain failures and broadcasts `manager:warning` after 3 consecutive failures. | `ensureExtraPathsOnProcessEnv`, `buildAgentEnv`, `getOAuthToken`, `invalidateOAuthToken`, `buildAgentEnvWithAuth`, `refreshOAuthTokenFromKeychain` |
| `paths.ts` | BDE data-directory constants — `BDE_DIR` (overridable via `BDE_DATA_DIR`), `BDE_DB_PATH` (overridable via `BDE_DB_PATH` or `BDE_TEST_DB`), and derived paths. Also exports repo config helpers and the default worktree bases: `DEFAULT_PIPELINE_WORKTREE_BASE` (`~/.bde/worktrees` — fallback when `agentManager.worktreeBase` is unset) and `ADHOC_WORKTREE_BASE` (`~/.bde/worktrees-adhoc` — shared between `adhoc-agent.ts` and `lib/review-paths.ts`). Both bases live under `~/.bde/` so all BDE state consolidates under one dotfile directory that's hidden from Finder and typically excluded by file-indexers that scan visible home subdirectories. | `BDE_DIR`, `BDE_DB_PATH`, `BDE_AGENTS_INDEX`, `BDE_MEMORY_DIR`, `DEFAULT_PIPELINE_WORKTREE_BASE`, `ADHOC_WORKTREE_BASE`, `validateWorktreeBase`, `validateTestDbPath`, `getConfiguredRepos`, `getRepoPath` |
| `async-utils.ts` | Promisified `execFile` and `sleep` | `execFileAsync`, `sleep` |
| `retry-constants.ts` | Named retry timing constants shared by service-layer dispatch operations | `DISPATCH_RETRY_DELAY_MS` |
| `patch-validation.ts` | Validates and filters patch content before applying | `validateAndFilterPatch` |
| `review-paths.ts` | Path validators for review git refs, worktree paths, and file paths. `validateWorktreePath` accepts paths under either the configured pipeline base (`agentManager.worktreeBase`) or the adhoc base (`ADHOC_WORKTREE_BASE`) — both are returned by `getAllowedWorktreeBases`. `assertWorktreeExists` throws `WorktreeMissingError` when the directory has been pruned, so review handlers fail with a clear message instead of the misleading `spawn git ENOENT` (which surfaces when `child_process.spawn(..., { cwd })` is given a missing cwd). | `validateGitRef`, `validateWorktreePath`, `validateFilePath`, `assertWorktreeExists`, `getWorktreeBase`, `getAllowedWorktreeBases`, `WorktreeMissingError` |
| `git-operations.ts` | Git branch/commit operations — rebase, push, fetch, ff-merge, auto-commit, squash merge. Moved from `agent-manager/` (consumed by services outside agent orchestration). `executeSquashMerge` now splits the merge/commit into two try-blocks with explicit `abortAndReset` rollback on either failure, verifies `git status --porcelain` after the commit, and uses `--no-commit` on the squash merge so the rollback boundary is unambiguous. | `rebaseOntoMain`, `pushBranch`, `fetchMain`, `ffMergeMain`, `autoCommitIfDirty`, `executeSquashMerge`, `SquashMergeOpts` |
| `main-repo-guards.ts` | State guards for the MAIN repo used during worktree setup and auto-merge. `assertRepoCleanOrAbort` refuses to proceed when `.git/MERGE_HEAD` exists or the working tree is dirty; attempts best-effort `git merge --abort` + `git checkout -- .` before throwing. `getMainRepoPorcelainStatus` returns `git status --porcelain` output, shared by tripwire callers in `run-agent.ts` and `executeSquashMerge`. | `assertRepoCleanOrAbort`, `getMainRepoPorcelainStatus` |
| `post-merge-dedup.ts` | Post-merge CSS deduplication orchestrator — identifies changed CSS files, deduplicates, and commits. Moved from `services/` | `runPostMergeDedup`, `DedupReport` |
| `prompt-composer.ts` | Central prompt dispatcher — routes `BuildPromptInput` to per-agent-type builders. Moved from `agent-manager/` | `buildAgentPrompt`, `BuildPromptInput`, `AgentType`, `classifyTask`, `TaskClass` |
| `resolve-dependents.ts` | Resolves blocked dependents when a task reaches terminal status. Cascade cancel is condition-aware: `condition: 'on_success'` triggers cancel on failure; `condition: 'always'` never cancels (unblocks regardless); legacy `type: 'hard'` (no condition) preserved for backward compat. Calls `onTaskTerminal` before `updateTask` so a throw prevents partial DB transitions. | `resolveDependents` |
| `validation.ts` | Input validation helpers for IPC handlers — safe identifier checks to prevent path traversal | `isValidAgentId`, `isValidTaskId` |
| `secure-storage.ts` | Wraps `electron.safeStorage` for encrypting sensitive settings — `ENC:` prefix distinguishes encrypted values from legacy plaintext. `encryptSetting()` throws if encryption is unavailable (never stores plaintext silently). Source: `src/main/secure-storage.ts` | `SENSITIVE_SETTING_KEYS`, `isEncryptionAvailable`, `encryptSetting`, `decryptSetting` |
| `settings.ts` | Facade over `data/settings-queries.ts` — plain string `getSetting`/`setSetting`/`deleteSetting` plus JSON-typed `getSettingJson<T>`/`setSettingJson<T>`. `getSettingJson` accepts an optional runtime type-guard `validator?: (u: unknown) => u is T`; when supplied, a shape mismatch returns `null` and logs a warning rather than letting a corrupt row propagate as `T`. Sensitive keys (e.g. `github.token`) are encrypted at rest via `secure-storage.ts`. Also exports well-known key constants and helpers `getMcpEnabled`, `getMcpPort`. Source: `src/main/settings.ts` | `getSetting`, `setSetting`, `deleteSetting`, `getSettingJson`, `setSettingJson`, `getMcpEnabled`, `getMcpPort`, `SETTING_DEPENDENCY_CASCADE_BEHAVIOR`, `SETTING_MCP_ENABLED`, `SETTING_MCP_PORT` |
