# Lib — Main

Utility functions and shared helpers for the main process.
Source: `src/main/lib/` (and the shared SDK streaming helper at `src/main/sdk-streaming.ts`).

| Module | Purpose | Key Exports |
|--------|---------|-------------|
| `../broadcast.ts` | **(Epic 9 / T-19)** Main-process → renderer event bus. `broadcast(channel, data)` sends immediately to all windows; `broadcastCoalesced(channel, payload)` batches rapid events into a 16ms flush to reduce IPC overhead. `CoalescedBroadcaster` class exported for unit testing with isolated instances — the module-level `_coalesced` singleton preserves all existing call sites. | `broadcast`, `broadcastCoalesced`, `CoalescedBroadcaster` |
| `../sdk-streaming.ts` | Shared Claude Agent SDK streaming helpers. `SdkStreamingOptions.model` is **required** — every call site must resolve it from `agents.backendConfig` via `resolveAgentRuntime(type).model`. No silent default. `permissionMode` and `allowDangerouslySkipPermissions` are per-call opt-ins (undefined → SDK default, which prompts) — the previously hard-coded `bypassPermissions` was removed so every caller relying on headless tool use must explicitly opt in beside a vetted narrow `tools` list. Timeout always throws `StreamTimeoutError` (carries `truncated: true` and `partialText`) — never returns a partial string with no signal. | `runSdkStreaming`, `runSdkOnce`, `SdkStreamingOptions`, `ToolUseEvent`, `StreamTimeoutError` |
| `env-utils.ts` | Environment setup — PATH augmentation (with optional `FLEET_EXTRA_PATHS` prefix), OAuth token management with symlink/size guards, agent environment building. `buildAgentEnv` also resolves and prepends the active `node` binary directory (via `resolveNodeExecutable`) so all agent spawn paths (adhoc, pipeline, workbench) can find `node` in packaged `.app` builds launched from Finder. `buildWorktreeEnv(worktreePath)` extends the base env by prepending `<worktreePath>/node_modules/.bin` when it exists — fixes `turbo: command not found` for workspace repos. `refreshOAuthTokenFromKeychain` tracks consecutive Keychain failures and broadcasts `manager:warning` after 3 consecutive failures. | `ensureExtraPathsOnProcessEnv`, `buildAgentEnv`, `buildWorktreeEnv`, `getOAuthToken`, `invalidateOAuthToken`, `buildAgentEnvWithAuth`, `refreshOAuthTokenFromKeychain` |
| `paths.ts` | FLEET data-directory constants — `FLEET_DIR` (overridable via `FLEET_DATA_DIR`), `FLEET_DB_PATH` (overridable via `FLEET_DB_PATH` or `FLEET_TEST_DB`), and derived paths. Also exports repo config helpers and the default worktree bases: `DEFAULT_PIPELINE_WORKTREE_BASE` (`~/.fleet/worktrees` — fallback when `agentManager.worktreeBase` is unset) and `ADHOC_WORKTREE_BASE` (`~/.fleet/worktrees-adhoc` — shared between `adhoc-agent.ts` and `lib/review-paths.ts`). Both bases live under `~/.fleet/` so all FLEET state consolidates under one dotfile directory that's hidden from Finder and typically excluded by file-indexers that scan visible home subdirectories. | `FLEET_DIR`, `FLEET_DB_PATH`, `FLEET_AGENTS_INDEX`, `FLEET_MEMORY_DIR`, `DEFAULT_PIPELINE_WORKTREE_BASE`, `ADHOC_WORKTREE_BASE`, `RepoConfig`, `validateWorktreeBase`, `validateTestDbPath`, `getConfiguredRepos`, `getRepoConfig`, `getRepoPath`, `getRepoPromptProfile`, `getRepoEnvVars` |
| `async-utils.ts` | Promisified `execFile` and `sleep` | `execFileAsync`, `sleep` |
| `default-branch.ts` | Resolves a repo's default branch by reading `origin/HEAD` (`git symbolic-ref refs/remotes/origin/HEAD`). Caches per-repoPath for the lifetime of the process. Falls back to `'main'` when `origin/HEAD` is not configured. Used by every git operation that previously hardcoded `origin/main` (rev-list, fetch, ff-merge, rebase, diff) so repos using `master`/`develop`/etc. work end-to-end. | `resolveDefaultBranch`, `clearDefaultBranchCache` |
| `retry-constants.ts` | Named retry timing constants shared by service-layer dispatch operations | `DISPATCH_RETRY_DELAY_MS` |
| `patch-validation.ts` | Filters a patch to only include keys present in the allowlist. **Epic 6:** signature updated to accept `ReadonlySet<string>` (previously `Set<string>`) — callers can now pass `UPDATE_ALLOWLIST_SET` without a cast. | `validateAndFilterPatch` |
| `review-paths.ts` | Path validators for review git refs, worktree paths, and file paths. `validateWorktreePath` accepts paths under either the configured pipeline base (`agentManager.worktreeBase`) or the adhoc base (`ADHOC_WORKTREE_BASE`) — both are returned by `getAllowedWorktreeBases`. `assertWorktreeExists` throws `WorktreeMissingError` when the directory has been pruned, so review handlers fail with a clear message instead of the misleading `spawn git ENOENT` (which surfaces when `child_process.spawn(..., { cwd })` is given a missing cwd). | `validateGitRef`, `validateWorktreePath`, `validateFilePath`, `assertWorktreeExists`, `getWorktreeBase`, `getAllowedWorktreeBases`, `WorktreeMissingError` |
| `git-operations.ts` | Git branch/commit operations — rebase, push, fetch, ff-merge, auto-commit, squash merge. Moved from `agent-manager/` (consumed by services outside agent orchestration). All branch operations resolve the repo's default branch dynamically via `resolveDefaultBranch` instead of hardcoding `main` — works for repos defaulting to `master`/`develop`/etc. `executeSquashMerge` splits the merge/commit into two try-blocks with explicit `abortAndReset` rollback on either failure, verifies `git status --porcelain` after the commit, and uses `--no-commit` on the squash merge so the rollback boundary is unambiguous. `GIT_ARTIFACT_PATTERNS` includes `'node_modules'` (no trailing slash) as a belt-and-suspenders guard so `stageWithArtifactCleanup` unstages the symlink even if `.git/info/exclude` did not prevent staging. | `rebaseOntoMain`, `pushBranch`, `fetchMain`, `ffMergeMain`, `autoCommitIfDirty`, `executeSquashMerge`, `SquashMergeOpts` |
| `main-repo-guards.ts` | State guards for the MAIN repo used during worktree setup and auto-merge. `assertRepoCleanOrAbort` refuses to proceed when `.git/MERGE_HEAD` exists or the working tree is dirty; attempts best-effort `git merge --abort` + `git checkout -- .` before throwing. `getMainRepoPorcelainStatus` returns `git status --porcelain` output, shared by tripwire callers in `run-agent.ts` and `executeSquashMerge`. All `execFileAsync` calls carry a 30 s timeout so a stalled git process cannot block the drain loop indefinitely. | `assertRepoCleanOrAbort`, `getMainRepoPorcelainStatus` |
| `post-merge-dedup.ts` | Post-merge CSS deduplication orchestrator — identifies changed CSS files, deduplicates, and commits. Moved from `services/` | `runPostMergeDedup`, `DedupReport` |
| `prompt-composer.ts` | Central prompt dispatcher — routes `BuildPromptInput` to per-agent-type builders. Moved from `agent-manager/` | `buildAgentPrompt`, `BuildPromptInput`, `AgentType`, `classifyTask`, `TaskClass` |
| `resolve-dependents.ts` | Resolves blocked dependents when a task reaches terminal status. Cascade cancel is condition-aware: `condition: 'on_success'` triggers cancel on failure; `condition: 'always'` never cancels (unblocks regardless); legacy `type: 'hard'` (no condition) preserved for backward compat. Calls `onTaskTerminal` before `updateTask` so a throw prevents partial DB transitions. | `resolveDependents` |
| `validation.ts` | Input validation helpers for IPC handlers — safe identifier checks to prevent path traversal | `isValidAgentId`, `isValidTaskId` |
| `secure-storage.ts` | Wraps `electron.safeStorage` for encrypting sensitive settings — `ENC:` prefix distinguishes encrypted values from legacy plaintext. `encryptSetting()` throws if encryption is unavailable (never stores plaintext silently). Source: `src/main/secure-storage.ts` | `SENSITIVE_SETTING_KEYS`, `isEncryptionAvailable`, `encryptSetting`, `decryptSetting` |
| `settings.ts` | Facade over `data/settings-queries.ts` — plain string `getSetting`/`setSetting`/`deleteSetting` plus JSON-typed `getSettingJson<T>`/`setSettingJson<T>`. `getSettingJson` accepts an optional runtime type-guard `validator?: (u: unknown) => u is T`; when supplied, a shape mismatch returns `null` and logs a warning rather than letting a corrupt row propagate as `T`. Sensitive keys (e.g. `github.token`) are encrypted at rest via `secure-storage.ts`. Also exports well-known key constants and helpers `getMcpEnabled`, `getMcpPort`. Source: `src/main/settings.ts` | `getSetting`, `setSetting`, `deleteSetting`, `getSettingJson`, `setSettingJson`, `getMcpEnabled`, `getMcpPort`, `SETTING_DEPENDENCY_CASCADE_BEHAVIOR`, `SETTING_MCP_ENABLED`, `SETTING_MCP_PORT` |
