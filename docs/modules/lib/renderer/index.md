# Lib — Renderer

Utility functions and shared helpers for the renderer process.
Source: `src/renderer/src/lib/`

| Module | Purpose | Key Exports |
|--------|---------|-------------|
| `copy-to-clipboard.ts` | Copy text to system clipboard with standard FLEET toast feedback. Shows "Copied to clipboard" on success, "Could not copy — please copy manually" on failure. Shared helper extracted from onboarding steps. | `copyToClipboard` |
| `render-markdown.ts` | Convert markdown to sanitized HTML safe for dangerouslySetInnerHTML. Includes DOMPurify `afterSanitizeAttributes` hook with a positive protocol allowlist (`https:`, `http:`, `mailto:` only) — any non-allowlisted protocol (including `data:`, `javascript:`, relative paths) has `href` removed regardless of exception-based fallback. **T-20:** `class` is no longer in `ALLOWED_ATTR` — attacker-authored markdown cannot inject `fleet-pulse` or any other CSS class. | `renderMarkdown` |
| `dashboard-types.ts` | Shared dashboard domain types used by stores and components. Owns `FeedEvent`, `DashboardStats`, `ChartBar`, and `EpicAccent` as canonical definitions — no neon imports, no local redeclarations in `CenterColumn`, `StatusRail`, `KPIStrip`, or `useDashboardMetrics`. | `FeedEvent`, `DashboardStats`, `ChartBar`, `EpicAccent` |
| `ringBuffer.ts` | Fixed-size circular buffer with O(1) push and no heap allocation after initialization. `createRingBuffer(size)` allocates once; `pushToRingBuffer(buf, item)` writes in place and advances the head pointer; `readRingBuffer(buf)` returns all stored items in insertion order. Used by `sprintEvents` store to cap per-agent event accumulation without array spreads. | `RingBuffer`, `createRingBuffer`, `pushToRingBuffer`, `readRingBuffer` |
| `optimisticUpdateManager.ts` | Pure functions for managing optimistic update state in the sprint tasks store. No Zustand dependency. `PendingUpdate.fields` is typed `readonly SprintTaskField[]` (alias for `keyof SprintTask`) so typoed field names are a compile error at every producer. `areSprintTasksEquivalent` does field-by-field equality used by the polling merge to preserve object identity without `JSON.stringify`-ing every task. | `mergePendingFields`, `expirePendingUpdates`, `trackPendingOperation`, `areSprintTasksEquivalent`, `PendingUpdate`, `PendingUpdates`, `SprintTaskField` |
| `task-status-ui.ts` | Backward-compatible re-export shim. `STATUS_METADATA`, `BucketKey`, and `StatusMetadata` now live in `src/shared/task-statuses.ts`; this file re-exports them for renderer callers. | `STATUS_METADATA`, `BucketKey`, `StatusMetadata` |
| `view-resolver.tsx` | Maps `View` keys to lazily-loaded React components. Moved from `components/layout/` to break the `layout↔panels` import cycle. Also exports `VIEW_LOADERS` for hover-based preloading. | `resolveView`, `VIEW_LOADERS` |
| `services/git.ts` | Git IPC adapter — wraps `window.api.git.*` for use by stores. `isConfiguredRepoPath(path)` lets callers pre-check whether a path is under a configured repo root, avoiding `git:status` rejections from the main-process boundary validator. `diffBetweenRefs` added (T-46) so review hooks can fetch incremental diffs through the service layer. | `getRepoPaths`, `isConfiguredRepoPath`, `getGitStatus`, `getGitDiff`, `stageFiles`, `unstageFiles`, `commit`, `push`, `getBranches`, `diffBetweenRefs` |
| `services/sprint.ts` | Sprint task IPC adapter — wraps `window.api.sprint.*`. **T-60:** Removed redundant `as Promise<...>` casts; types now inferred directly from the preload bridge declaration. `exportTaskHistory(taskId)` added so the store action can route through this adapter. **T-47:** `getLastPrompt(taskId)` added so Code Review's TopBar fetches the rendered agent prompt through the service layer rather than calling `window.api` directly. | `listTasks`, `updateTask`, `deleteTask`, `createTask`, `batchUpdate`, `generatePrompt`, `exportTaskHistory`, `getLastPrompt` |
| `services/review.ts` | **(T-59, new)** Code Review IPC adapter — wraps `window.api.review.*`. Single chokepoint for all renderer→main review traffic; hooks (`useSingleTaskReviewActions`, `useBatchReviewActions`, `useReviewChanges`, `useReviewCommits`, `useReviewFreshness`, `useReviewPartnerActions`, `useApproveAction`) import from here instead of touching `window.api.review.*` directly. | `getDiff`, `getCommits`, `getFileDiff`, `shipIt`, `shipBatch`, `mergeLocally`, `createPr`, `requestRevision`, `rebase`, `discard`, `markShippedOutsideFleet`, `approveTask`, `autoReview`, `chatStream`, `abortChat`, `onChatChunk`, `checkFreshness`, `checkAutoReview` |
| `services/groups.ts` | Task groups IPC adapter — wraps `window.api.groups.*` | `listGroups`, `getGroupTasks`, `createGroup`, `updateGroup`, `deleteGroup`, `addTask`, `removeTask`, `queueAll`, `reorderTasks`, `addDependency`, `removeDependency`, `updateDependencyCondition` |
| `services/settings-storage.ts` | Settings persistence IPC adapter — wraps `window.api.settings.*`. `getJsonSetting<T>(key, validate)` requires a runtime guard predicate; if the stored payload fails validation the function returns `null` and logs a warning rather than coercing the value through `as T`. | `getSetting`, `setSetting`, `getJsonSetting`, `setJsonSetting` |
| `services/agents.ts` | Agent management IPC adapter — wraps `window.api.agents.*` | `subscribeToAgentEvents`, `getAgentEventHistory`, `listAgents`, `readAgentLog`, `importAgent`, `getProcesses`, `spawnLocal`, `tailLog` |
| `services/dashboard.ts` | Dashboard/system/PR metrics IPC adapter — wraps `window.api.dashboard.*`, `window.api.system.*`, `window.api.pr.*` | `getCompletionsPerHour`, `getRecentEvents`, `getPrList`, `getDailySuccessRate`, `getLoadAverage` |
| `services/cost.ts` | Cost history IPC adapter — wraps `window.api.cost.*` | `getAgentCostHistory` |
| `view-registry.ts` | Single source of truth for view metadata (label, icon, shortcut, description). All 8 views are visible: Dashboard (⌘1), Agents (⌘2), IDE (⌘3), Task Pipeline (⌘4), Code Review (⌘5), Source Control (⌘6), Settings (⌘7), Task Planner (⌘8). Derives backward-compat exports `VIEW_LABELS`, `VIEW_ICONS`, `VIEW_SHORTCUTS`, `VIEW_SHORTCUT_MAP`. | `VIEW_REGISTRY`, `ViewMetadata`, `VIEW_LABELS`, `VIEW_ICONS`, `VIEW_SHORTCUTS`, `VIEW_SHORTCUT_MAP` |
| `shortcuts-data.ts` | Keyboard shortcut categories for the shortcuts overlay. `GLOBAL_SHORTCUTS` is derived from VIEW_REGISTRY (filtered to visible views with shortcuts) so view entries never drift out of sync. `IDE_SHORTCUTS` and `CODE_REVIEW_SHORTCUTS` are static. | `GLOBAL_SHORTCUTS`, `IDE_SHORTCUTS`, `CODE_REVIEW_SHORTCUTS`, `SHORTCUT_CATEGORIES`, `Shortcut`, `ShortcutCategory` |
| `wip-policy.ts` | WIP limit policy function. `canLaunchTask(activeCount, maxConcurrent)` returns `true` only when `activeCount < maxConcurrent`. Single source of truth for the WIP limit rule — used by the sprint tasks store and enforced at the drain loop. | `canLaunchTask` |
| `utils.ts` | Shared renderer utilities. `validateGitHubUrl(url)` returns the URL only when it parses as `https://github.com/...` (or `www.github.com`); otherwise `null`. Use it to gate any `href` or `window.open` against attacker-supplied PR URLs. | `cwdToRepoLabel`, `withAlpha`, `validateGitHubUrl` |
| `extract-test-runs.ts` | Extracts test command runs from agent event streams for display in the Verification tab. Pattern-matches `agent:tool_call` Bash events against `TEST_COMMAND_PATTERN` (npm/yarn/pnpm/vitest/jest/pytest/cargo/go test, plus gradle and maven variants). Two alternation groups: `./`-prefixed commands use `(?:^|[\s;|&])` anchoring; all others use `\b` word boundaries. Pairs each matched call with the next `agent:tool_result` to produce `TestRun` objects. | `extractTestRuns`, `TestRun` |
| `format.ts` | Pure formatting utilities for the renderer. `repoColor(repoName, repos)` and `repoBadgeVariant(repoName, repos)` both accept an explicit `RepoOption[]` list — no hardcoded repo names. Badge variant derived from index-based cycle (0→info, 1→warning, 2→success). Also exports `timeAgo`, `formatElapsed`, `formatDuration`, `formatDurationMs`, `modelBadgeLabel`, `formatDate`, `formatTime`, `formatTokensCompact`. | `timeAgo`, `formatElapsed`, `formatDuration`, `formatDurationMs`, `modelBadgeLabel`, `repoBadgeVariant`, `repoColor`, `formatDate`, `formatTime`, `formatTokens`, `formatTokensCompact` |
| `task-format.ts` | Sprint pipeline formatting helpers. `getDotColor` maps task status + pr_status to a CSS variable. `failureCategoryForReason` maps a task's `failure_reason` string to a `{ label, colorClass }` chip descriptor used by `TaskPill` and `TaskDetailDrawer`. | `getDotColor`, `failureCategoryForReason`, `FailureCategory` |
| `partitionSprintTasks.ts` | Partitions sprint tasks into 8 mutually exclusive buckets. `pendingReview` holds `status='review'` tasks; `openPrs` holds `active + pr_status='open'|'branch_only'` tasks. All other statuses route via `STATUS_METADATA[status].bucketKey`. `done` bucket is sorted by `completed_at` descending. | `partitionSprintTasks`, `SprintPartition` |
| `spec-quality.ts` | Pure function `analyzeSpecQuality(spec)` that detects whether a spec markdown contains the three required sections (Files to change / How to test / Acceptance criteria). Returns `SpecQuality` — used by `PlSpecPane` quality pills. | `analyzeSpecQuality`, `SpecQuality` |
| `spec-diff.ts` | Pure function to build synthetic unified diffs for before/after text comparisons. Extracted from `EditDiffCard.tsx` for reuse by spec diffing components. | `buildSyntheticDiff` |
| `task-status.ts` | Maps a sprint task `status` + `pr_status` pair to a `StatusDotKind` for use by `StatusDot`. Shared between `TaskDetailDrawerV2` and `TaskRowV2` to eliminate duplication. | `statusToDotKind` |
| `documentation-data.ts` | Structured documentation content for the Settings → Documentation tab and the `meta.specGuidelines` MCP tool. Defines `DocTopic[]` with five topics: Epics, Writing a Task Spec, Task Dependencies, Agent Types, MCP Integration. Also exports `renderToMarkdown()` for the MCP tool. | `DOCUMENTATION_TOPICS`, `DocTopic`, `DocSection`, `DocBadge`, `renderToMarkdown` |
| `monaco-theme.ts` | Builds the legacy `fleet-dark` and `fleet-light` Monaco syntax themes from `--fleet-*` CSS variables. Defines syntax token rules (keywords, strings, comments, numbers, functions, types) plus full editor chrome (background, gutter, line numbers, selection, widgets, scrollbars). | `getMonacoTheme`, `getLightMonacoTheme` |
| `monaco-theme-v2.ts` | Builds the V2 chrome-only Monaco theme (`fleet-v2-dark` / `fleet-v2-light`). Reads V2 design tokens via `getComputedStyle(document.documentElement)` and maps them to `editor.background`, `editorGutter.background`, `editor.lineHighlightBackground`, `editor.selectionBackground`, and the line-number colors. `inherit: true` + empty `rules` means syntax token colors fall through to whichever `fleet-*` theme is currently active. Coerces `rgba(…)` token values (e.g. `--accent-soft`) to `#RRGGBBAA` hex which is the format Monaco accepts. | `getMonacoV2Theme`, `V2_THEME_DARK`, `V2_THEME_LIGHT` |
| `describeAgentStep.ts` | Converts agent event stream messages into human-readable step descriptions for the dashboard's ActiveAgentsCard. `describeAgentStep(event)` maps SDK message types (`tool_call`, `tool_result`, `completion`) to concise past-tense phrases. Used by the dashboard to label agent progress without storing step descriptions in the database. | `describeAgentStep` |
| `sanitize-agent-output.ts` | Guards for agent-supplied text before it reaches the UI or database. Provides `sanitizeAgentPayloadString` (truncate + strip FLEET XML boundary tags via a targeted allowlist — only the 24 known FLEET boundary tags are removed, leaving legitimate HTML like `<pre>` and `<code>` intact) and `stripActionMarkers` (remove `[ACTION:…]` markers from chat bubbles). Named constants `MAX_TASK_TITLE_CHARS` (500) and `MAX_TASK_SPEC_CHARS` (8000) are the single source of truth for these limits in the renderer. | `sanitizeAgentPayloadString`, `stripActionMarkers`, `MAX_TASK_TITLE_CHARS`, `MAX_TASK_SPEC_CHARS` |
