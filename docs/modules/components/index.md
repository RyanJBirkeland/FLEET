# Components

React UI components, organized by domain group.
Source: `src/renderer/src/components/`

| Module | Group | Purpose | Key Exports |
|--------|-------|---------|-------------|
| `ReviewQueue.tsx` | code-review | Displays tasks awaiting review with keyboard navigation (j/k). Scoped store subscription via `useShallow` to avoid re-renders on unrelated task changes. Select-all checkbox has `aria-label` with task count; visually-hidden `aria-live="polite"` region announces batch selection changes. | `ReviewQueue` |
| `CommandPalette.tsx` | layout | Command palette with fuzzy search and action execution. Spawns assistant using the first configured repository path. | `CommandPalette` |
| `Sidebar.tsx` | layout | Persistent nav sidebar with view badges for review/failed counts. Uses named selectors from sprintTasks store. | `Sidebar` |
| `AgentCard.tsx` | agents | Compact card showing agent status, cost, and duration. Uses `useBackoffInterval` for the live duration ticker. | `AgentCard` |
| `ConsoleHeader.tsx` | agents | 56px glass header for AgentConsole — status dot, task name, model badge, duration, phase, cost, and ctx token counter. Ctx counter calls `window.api.agents.getContextTokens` once on mount and polls every 3s while running; displays the latest-turn context window size live, and the peak across the run for finished agents. Action buttons: open terminal, stop (running), promote to Code Review (adhoc with worktree), copy log. | `ConsoleHeader` |
| `CreateEpicModal.tsx` | planner | Form modal for creating new epics (task groups). Fields: name (required), icon, goal, accent color. Template selection supported — templates populate fields and their tasks. Repo selection defaults to the first configured repo via `useRepoOptions()` — no hardcoded repo list. | `CreateEpicModal` |
| `EpicHeader.tsx` | planner | Header region of the epic detail panel. Renders epic icon, name, goal, Ask AI button (when `totalCount > 0`), overflow menu (edit/ready/complete/delete), and a 3px progress stripe (`doneCount/totalCount`). Props: `group`, `isReady`, `isCompleted`, `doneCount`, `totalCount`, `onOpenAssistant`, `onEdit`, `onToggleReady`, `onMarkCompleted`, `onDelete`. | `EpicHeader`, `EpicHeaderProps` |
| `EpicDetail.tsx` | planner | Full detail panel for a selected epic. Computes `doneCount`/`totalCount` from tasks and passes them to `EpicHeader`. Accepts `onOpenAssistant: () => void` prop and threads it through to `EpicHeader`. Also renders `EpicProgress`, `EpicDependencySection`, `TaskList`, and the queue bar. | `EpicDetail`, `EpicDetailProps` |
| `WorkbenchPanel.tsx` | planner | Slide-over panel that embeds `TaskWorkbench` inside Task Planner. Opens via "Add Task" / "Edit Task" actions in PlannerView instead of navigating to the standalone Task Workbench view. | `WorkbenchPanel`, `WorkbenchPanelProps` |
| `PlannerAssistant.tsx` | planner | AI planning assistant drawer that slides over the epic detail panel. Streams chat via `workbench:chatStream`; parses `[ACTION:...]` markers from assistant replies to surface create-task, create-epic, and update-spec action cards inline. Exported `parseActionMarkers` is unit-tested. | `PlannerAssistant`, `PlannerAssistantProps`, `parseActionMarkers`, `ParseResult` |
| `WorkbenchForm.tsx` | task-workbench | Task creation/edit form with AI copilot, dependency picker, and validation checks. Repo dropdown is populated dynamically via `useRepoOptions()` — no hardcoded repo list. Priority and Dependencies are always visible; Advanced section contains cost, playground, and cross-repo contract. Per-task model selection was removed once Settings → Models became the single routing source. | `WorkbenchForm` |
| `SpecEditor.tsx` | task-workbench | Spec textarea with template buttons (Feature, Bug Fix, Refactor, Test) sourced from `DEFAULT_TASK_TEMPLATES` in `src/shared/constants.ts`, Generate Spec, Research Codebase, and quality hint indicators. | `SpecEditor` |
| `BatchActionsToolbar.tsx` | code-review | Renders the batch action buttons (Merge All, Ship All, Create PRs, Discard All, Clear) with in-flight spinner state. Extracted from `TopBar` to eliminate JSX duplication. Count span has `aria-live="polite"`; the 4 batch action buttons (Merge All, Ship All, Create PRs, Discard All) each have an `aria-label` that includes the selection count (the Clear button does not). | `BatchActionsToolbar` |
| `ConnectionsSection.tsx` | settings | Settings section for auth status, GitHub credential management, webhooks, and local MCP server. Displays Claude Code CLI auth status (CLI found, token status, expiry). Shows system encryption availability via green/red encryption status banner. Includes a "Read-only mode (skip GitHub)" checkbox that toggles `settings.githubOptedOut` so users can enter or leave opted-out mode without re-running onboarding. Embeds `WebhooksSection` and `LocalMcpServerSection` at the bottom. | `ConnectionsSection` |
| `LocalMcpServerSection.tsx` | settings | Settings card for the local MCP server — enable/disable toggle, port input, bearer token display/copy/regenerate, and "Copy Claude Code config" button that writes the JSON snippet to clipboard. Reads `mcp.enabled` and `mcp.port` from settings on mount; fetches the bearer token via `mcp:getToken`. Regenerate confirms with the user before calling `mcp:regenerateToken` and reveals the new token. | `LocalMcpServerSection` |
| `AgentManagerSection.tsx` | settings | Settings section for pipeline configuration (concurrency, worktree base, runtime, auto-start) and agent permissions. Embeds `AgentPermissionsSection` below the pipeline config card. Model selection lives in Settings → Models — not here. | `AgentManagerSection` |
| `AgentPermissionsSection.tsx` | settings | Agent tool permission editor — consent banner, preset buttons (Recommended/Restrictive/Permissive), allow-tool grid with names + descriptions, and custom deny-rule list. Reads/writes `~/.claude/settings.json` via `claudeConfig` IPC. | `AgentPermissionsSection` |
| `AppearanceSection.tsx` | settings | Settings section for theme, accent color, tear-off window behavior, notifications, and keyboard shortcuts. Embeds `NotificationsSection` and `KeybindingsSettings`. | `AppearanceSection` |
| `AboutSection.tsx` | settings | Settings section for app version, source link, and API usage history. Embeds `CostSection`. | `AboutSection` |
| `LocalMcpServerSection.tsx` | settings | Settings section for the Local MCP Server. Shows enabled/disabled toggle, default port display (18792), and copy-to-clipboard action for the bearer token stored in `~/.bde/mcp-token`. | `LocalMcpServerSection` |
| `ReviewActionsBar.tsx` | code-review | Review action buttons (Ship It, Merge Locally, Create PR, Revise, Discard) plus freshness badge and rebase button. Renders in `full` or `compact` variant (compact uses a render-prop). Ship It button has `aria-busy` and contextual `aria-label` during in-flight state. | `ReviewActionsBar`, `ReviewActionCallbacks` |
| `VirtualizedDiffContent.tsx` | diff | Virtualized rendering of diff rows (file headers, hunk headers, lines). Manages scroll/viewport via ResizeObserver and binary-search visibility window. Exports shared row types (`FlatRow`, `HunkAddress`) and height constants used by `DiffViewer`. | `VirtualizedDiffContent`, `FlatRow`, `HunkAddress`, `rowHeight`, `ROW_HEIGHT`, `FILE_HEADER_HEIGHT`, `HUNK_HEADER_HEIGHT` |
| `VirtualizedDiffBanner.tsx` | diff | Banner shown above large diffs in virtualized mode, with a "Load full diff" button to disable virtualization and enable commenting. | `VirtualizedDiffBanner` |
| `GhStep.tsx` | onboarding | Onboarding step that checks gh CLI availability and authentication via `onboarding:checkGhCli` IPC. Shows two check rows: "gh CLI installed" and "gh CLI authenticated". Now surfaces a copy-pasteable `gh auth login` command (with a copy button) when the user is not authenticated, and exposes a "Skip — read-only mode" button that writes `settings.githubOptedOut = 'true'`. The primary "Next" button writes `'false'` so re-running onboarding can recover a user who opted out earlier. | `GhStep` |
| `AuthStep.tsx` | onboarding | Onboarding step that checks Claude Code CLI auth via `auth.status` IPC. Now wraps the check in a 10 s `withTimeout` — on timeout shows a `role="alert"` fallback ("check timed out, try Check Again or continue anyway"). When CLI is missing, surfaces copy-pasteable install command plus a link to the Claude docs; when CLI is present but token is missing, surfaces copy-pasteable `claude login`. Per-check icons carry `aria-label`s so screen readers announce pass/fail. | `AuthStep` |
| `GitStep.tsx` | onboarding | Onboarding step that checks git availability via `git.checkInstalled` IPC (runs `git --version`). Shows check status icon. Next button disabled while checking or when git is unavailable. | `GitStep` |
| `Onboarding.tsx` | (root) | First-launch setup check screen. Runs required checks (CLI, token, expiry, git) and optional checks (repos). Auto-advances when all required checks pass. Bypass button is **always** enabled (not `checking`) — the old "disabled-when-failed" trap is gone. Label becomes "Continue without `<check>`" (or "Continue without N required checks") so users know exactly what they're skipping, and a `role="alert"` risk callout names the skipped items with a "fix in Settings" pointer. Retry button renamed "Re-check" with explicit "after running the command, click Re-check to verify" guidance next to the instruction code block. | `Onboarding` |
| `PreloadBridgeError.tsx` | (root) | Fallback UI rendered from `main.tsx` when `window.api` is undefined at startup (preload script failed to load). Shows actionable recovery steps: right-click → Open, `xattr -dr com.apple.quarantine /Applications/BDE.app`, or reinstall. Pure presentational — no hooks or IPC. | `PreloadBridgeError` |
| `PlaygroundModal.tsx` | agents | Full-screen modal for previewing agent-generated playground files (HTML, SVG, Markdown, JSON). HTML and SVG use sandboxed iframe with DOMPurify sanitization; Markdown and JSON rendered as `<pre>` text. Editable source pane with syntax-highlight overlay. Re-sanitizes user edits via `playground:sanitize` IPC before rendering. Supports Split/Preview/Source modes. Scripts-enabled badge shown only for HTML. | `PlaygroundModal`, `PlaygroundModalProps` |
| `PlaygroundCard.tsx` | agents | Compact card for playground previews in agent chat. Shows filename, file size, and content-type badge (HTML/SVG/Markdown/JSON). Click to open PlaygroundModal. | `PlaygroundCard`, `PlaygroundCardProps` |
| `render-agent-markdown.tsx` | agents | Lightweight markdown-to-JSX renderer for agent-authored text. Handles `**bold**`, `` `code` ``, and `##`/`###` headings. Colocated CSS (`render-agent-markdown.css`) provides the `.console-md-*` primitives. Imported wherever agent messages render (agent console, SpecPanel, BDE Advisor). | `renderAgentMarkdown` |
| `PipelineHeader.tsx` | sprint | Pipeline view header with task stat badges, density toggle, export menu, and WIP slot capacity badge. Renders agent concurrency status (active/max slots) polled every 5s from agent manager. Stat badges are clickable to filter the pipeline. | `PipelineHeader` |
| `TaskDetailDrawer.tsx` | sprint | Task detail side panel for the Sprint Pipeline. Shows status, dependencies, spec/prompt links, agent activity preview, and PR info. The failure block is a labelled `<section>` ("Failure details" / "Cancellation details") shown for `failed`/`error`/`cancelled` tasks — renders `failure_reason` as a `<pre>` with a danger accent, plus the prior diagnostic notes and last three agent error events. Includes an "Override" section with operator escape-hatches: "Mark Failed" (visible for `queued`/`active`/`blocked` tasks, prompts for optional reason, calls `sprint.forceFailTask`) and "Force Done" (visible for any non-`done` status, danger-variant confirmation, calls `sprint.forceDoneTask` with `force: true`). | `TaskDetailDrawer`, `TaskDetailDrawerProps` |
| `TaskDetailActionButtons.tsx` | sprint | Status-aware action bar rendered inside `TaskDetailDrawer`. Renders Retry (primary) for any `failed`/`error`/`cancelled` task when `onRetry` is provided; otherwise renders the status-appropriate mix of Launch/Edit/Stop/Unblock/Clone/Delete/Export buttons. All buttons carry `aria-busy` + spinner during async handlers; Retry carries a task-named `aria-label`. | `TaskDetailActionButtons` |
| `DoneStep.tsx` | onboarding | Final onboarding wizard step. With multiple repos configured, renders a `<select>` labelled "Start in repository" — whatever the user picks is used as the first-task repo (no longer hardcoded to `repoOptions[0]`). With no repos configured, the CTA swaps to "Add a repository" which opens Settings; "Create your first task" is hidden in that state to avoid asking the user to create a task against an empty repo list. | `DoneStep` |
| `WelcomeStep.tsx` | onboarding | First step of the onboarding wizard — introduces BDE and links forward. Back button is wired to the `onBack` prop (formerly a no-op stub); it still only renders when `isFirst` is false, so the default first-step render is unchanged. | `WelcomeStep` |
| `OnboardingWizard.tsx` | onboarding | Multi-step onboarding wizard container. Backdrop carries `role="dialog" aria-modal="true"`. The progress rail is now an `<ol role="progressbar">` with `aria-valuemin/max/now/text`; each step `<li>` gets `aria-current="step"` when active plus a descriptive `aria-label` (e.g. "Step 2 of 6: Authentication (completed)"). | `OnboardingWizard` |
| `GitHubOptedOutBanner.tsx` | (root) | Small warning banner that appears in Task Workbench and Code Review when `settings.githubOptedOut === 'true'`. Session-dismissible via `sessionStorage` (`bde:github-opted-out-dismissed`) — re-appears on next launch until the user runs `gh auth login` and unsets the setting. Links to the onboarding copy-path to recover. | `GitHubOptedOutBanner` |
| `FloatingAgentButton.tsx` | floating-agent | Persistent FAB (bottom-right, z-index 900) that opens/closes the BDE Advisor panel. Shows neon glow when active. | `FloatingAgentButton` |
| `FloatingAgentPanel.tsx` | floating-agent | Slide-over chat panel for the BDE Advisor. Spawns adhoc agent sessions via `spawnLocal`, resumes via `steer`. Streams `agent:text` events into message bubbles. ⌘. and ↺ button reset the session. Token usage bar shows ~50k limit. | `FloatingAgentPanel` |
| `FloatingAgentMessage.tsx` | floating-agent | Message bubble for user/assistant turns in the floating agent panel. User bubbles right-aligned (plain text), assistant bubbles left-aligned and rendered via `renderAgentMarkdown` for bold/code/headings. | `FloatingAgentMessage` |
| `ModelsSection.tsx` | settings | Single "Active routing" card listing all six agent-type rows (Pipeline, Synthesizer, Copilot, Assistant, Adhoc, Reviewer). Each row's model picker is always interactive; the Local backend radio is enabled only for rows whose `supportsLocal` meta is true (Pipeline today) and carries an explanatory tooltip elsewhere. Loads and saves `agents.backendConfig` atomically. | `ModelsSection` |
| `LaunchpadGrid.tsx` | agents | Agent session launcher (tiles + repo chip + prompt input). The runtime model is resolved from `agents.backendConfig` and configured in Settings → Models — there is no in-launchpad model picker. | `LaunchpadGrid` |
