# Services

Business logic modules. IPC handlers delegate to these — they contain no business logic themselves.
Source: `src/main/services/`

| Module | Purpose | Key Exports |
|--------|---------|-------------|
| `repo-search-service.ts` | Grep-based codebase search for workbench research — literal fixed-string search to prevent ReDoS | `searchRepo`, `parseGrepOutput`, `RepoSearchResult`, `RepoSearchMatch` |
| `adhoc-promotion-service.ts` | Validates and promotes a completed adhoc agent worktree into the Code Review queue as a sprint task | `promoteAdhocToTask`, `PromoteAdhocParams`, `PromoteAdhocResult` |
| `webhook-delivery-service.ts` | Constructs and delivers test webhook events including HMAC-SHA256 signing and HTTP POST delivery | `deliverWebhookTestEvent`, `WebhookTestResult` |
| `checkpoint-service.ts` | Creates a git commit snapshot of the current agent worktree state without stopping the agent | `createCheckpoint`, `CheckpointResult` |
| `cost-queries.ts` (`src/main/`) | Thin wrapper delegating to `data/cost-queries`. All functions accept optional `db?: Database.Database` for injection. | `getCostSummary`, `getRecentAgentRunsWithCost`, `getAgentHistory` |
| `settings.ts` (`src/main/`) | Facade over `data/settings-queries` that transparently encrypts/decrypts sensitive keys (`SENSITIVE_SETTING_KEYS`) via `secure-storage.ts`. Lazy re-encryption of legacy plaintext values on first read (skipped with a warning if encryption unavailable). `setSetting()` will throw if encryption is unavailable for sensitive keys. All functions accept optional `db?: Database.Database` for injection. | `getSetting`, `setSetting`, `deleteSetting`, `getSettingJson`, `setSettingJson` |
| `agent-history.ts` (`src/main/`) | Agent run metadata and log management — persistent storage in SQLite agent_runs table. All exported functions accept optional `db?: Database.Database` for injection. | `listAgents`, `createAgentRecord`, `appendLog`, `readLog`, `getAgentMeta`, `updateAgentMeta`, `pruneOldAgents`, `hasAgent`, `findAgentByPid`, `listAgentRunsByTaskId`, `finalizeStaleAgentRuns`, `reconcileRunningAgentRuns`, `finalizeAllRunningAgentRuns`, `backfillUtcTimestamps` |
| `review-service.ts` | Orchestrates reviewer SDK calls, diff fetching, result caching, and result aggregation for auto-review | `ReviewService`, `ReviewServiceDeps`, `createReviewService`, `WorktreeMissingError` |
| `review-response-parser.ts` | Parses and validates raw reviewer model output into a typed `ParsedReview` shape | `parseReviewResponse`, `MalformedReviewError`, `ParsedReview` |
| `batch-import.ts` | Validates and bulk-creates sprint tasks from an import payload. Validates status values against `TASK_STATUSES` and repo names against the configured repo list. | `batchImportTasks`, `BatchImportResult` |
| `workflow-engine.ts` | Instantiates a workflow template as a set of linked sprint tasks | `instantiateWorkflow` |
| `sprint-mutation-broadcaster.ts` | Notification orchestration for sprint task mutations — in-process listeners, debounced `sprint:externalChange` IPC broadcast (200ms), and webhook dispatch | `notifySprintMutation`, `onSprintMutation`, `SprintMutationEvent`, `SprintMutationListener` |
| `spec-quality/validators/sync-validators.ts` | Synchronous spec validators — `SizeWarningsValidator` blocks specs >1000 words (error) and warns at >500; `RequiredSectionsValidator`, `FilePathsValidator`, `NumberedStepsValidator`, `BannedPhrasesValidator` | `SizeWarningsValidator`, `RequiredSectionsValidator`, `FilePathsValidator`, `NumberedStepsValidator`, `BannedPhrasesValidator` |
| `spec-quality/validators/prescriptiveness-validator.ts` | Async AI-based check for exploration language in Implementation Steps — returns error (not warning) when design decisions detected; returns `PRESCRIPTIVENESS_CHECK_FAILED` warning on SDK failure instead of silently passing | `PrescriptivenessValidator` |
| `task-terminal-service.ts` | Unified handler for all task terminal-status paths — records metrics, schedules dependency resolution (via `scheduleDependencyResolution`), and schedules worktree cleanup. All terminal paths converge here. Accepts optional `runInTransaction` dep; when provided (wired via `index.ts`), wraps the full cascade-cancellation loop in a single SQLite transaction for atomicity. | `TaskTerminalService`, `createTaskTerminalService`, `TaskTerminalServiceDeps` |
| `pr-poller.ts` (`src/main/`) | Polls GitHub for open PRs on all configured repos every 60s, fetches check-run summaries, and broadcasts `pr:listUpdated`. Uses `Promise.allSettled` so one failing repo does not abort the rest. Includes exponential backoff on poll errors. | `startPrPoller`, `stopPrPoller`, `getLatestPrList`, `refreshPrList`, `POLL_INTERVAL_MS` |
| `sprint-pr-poller.ts` (`src/main/`) | Polls PR statuses for sprint tasks with `pr_status=open` every 60s and auto-transitions tasks when PRs merge or close. DI-based via `createSprintPrPoller`; legacy `startSprintPrPoller` wraps it. | `createSprintPrPoller`, `startSprintPrPoller`, `stopSprintPrPoller`, `SprintPrPollerDeps`, `SprintPrPollerInstance` |
| `github-pr-status.ts` (`src/main/`) | Fetches individual PR status (merged/state/mergeableState) via GitHub REST API. `pollPrStatuses` uses `Promise.allSettled` so one failing PR fetch does not abort the rest. | `pollPrStatuses`, `PrStatusInput`, `PrStatusResult` |
| `bootstrap.ts` (`src/main/`) | App startup utilities — DB initialization, file watcher, CSP, cleanup tasks, PR pollers, background services. Tracks non-trivial async startup errors and surfaces them to the renderer via `emitStartupWarnings()` (call at `ready-to-show`). Trivial conditions (missing credentials, already-skipped imports) are filtered out. `validateRepoPaths()` checks each configured repo's `localPath` with `existsSync` at startup and queues a warning toast for any paths that don't exist on this machine. `warnPlaintextSensitiveSettings()` proactively re-encrypts any sensitive settings (github.token, supabase.serviceKey) found without the `ENC:` prefix — no longer just warns, now fixes silently or warns only on failure. | `initializeDatabase`, `startDbWatcher`, `setupCSP`, `setupCleanupTasks`, `startPrPollers`, `startBackgroundServices`, `emitStartupWarnings`, `warnPlaintextSensitiveSettings`, `validateRepoPaths`, `DEBOUNCE_MS` |
| [`credential-service.ts`](./credential-service.md) | Unified Claude + GitHub credential resolution with discriminated-return shape. Primary entry point for credential checks. `oauth-checker.checkOAuthToken` and `auth-guard.ensureSubscriptionAuth` now delegate here so every call site produces the canonical `CREDENTIAL_GUIDANCE` actionable message. Caches success 5 min, failure 30 s. Used by `spawn-and-wire.ts` for pre-spawn refresh (V0.6), `pr-operations.ts` for the `gh` auth guard, and the drain-loop precondition. | `createCredentialService`, `getDefaultCredentialService`, `CredentialService`, `CredentialResult`, `CredentialKind`, `CREDENTIAL_GUIDANCE`, `ClaudeCredentialStore`, `GithubCredentialStore` |
| `status-server.ts` | Read-only HTTP server exposing agent-manager status on `127.0.0.1:18791/status`. Now broadcasts `manager:warning` on bind failure with a specific message when the error is `EADDRINUSE` (points at "another BDE instance may be running"). | `createStatusServer`, `StatusServer` |
| `logger.ts` (`src/main/`) | File-backed logger that writes to `~/.bde/bde.log` with rotation at 10 MB (keeps 3 generations). Mirror writes to `console.*` are gated by `CONSOLE_LOG_ENABLED` — off in packaged builds (`NODE_ENV === 'production'`) unless the user sets `BDE_CONSOLE_LOG=1`. Saves ~397 dual-writes per startup in production. File writes always run. | `createLogger`, `Logger`, `logError` |
| `index.ts` (`src/main/`) | Electron main entry. `createWindow()` wraps `new BrowserWindow(…)` in try/catch and shows an error dialog + `app.quit()` on failure, preventing the silent hang that used to happen on headless systems or when Electron could not initialize a graphics context. | `createWindow` (module-private; no runtime exports) |
