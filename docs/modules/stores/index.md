# Stores

Zustand state stores. One store per domain concern.
Source: `src/renderer/src/stores/`

| Module | Purpose | Key Exports |
|--------|---------|-------------|
| `sprintTasks.ts` | Primary sprint task store — CRUD, optimistic updates, polling merge. `MUTABLE_TASK_FIELDS` now includes `worktree_path`, `fast_fail_count`, `duration_ms`, and `promoted_to_review_at` so `stableTaskRef` detects review-transition field changes and refreshes Code Review with current data. `PENDING_UPDATE_TTL` (5s) exported for test use. `stableTaskRef` preserves object identity when no mutable field changed, preventing unnecessary re-renders. `pollError` / `clearPollError()` used by SprintPipeline error banner. `selectActiveTasks` selector exported at store level so `SidebarV2` subscribes via `useShallow` without a per-module filter. `launchTask` removed from the store — agent launch logic lives exclusively in `useLaunchTask` hook; callers go through `useSprintTaskActions`. **T-17/T-18/T-19/T-20/T-21:** `sanitizeIncomingTasks` now accepts `SprintTask[]` (not `unknown[]`); `mergeSseUpdate` validates each field type before applying (invalid SSE payloads cannot corrupt task objects); `taskListsAreEqual` replaces `buildFingerprint`/sort for O(n) change detection on every background poll; `hasLoadedOnce` flag distinguishes initial load from background refresh. `retryTask(id)` wraps `sprint:retry` IPC and refreshes store state — the single canonical retry path for all UI surfaces. | `useSprintTasks`, `selectTasks`, `selectActiveTasks`, `selectActiveTaskCount`, `selectReviewTaskCount`, `selectFailedTaskCount`, `MUTABLE_TASK_FIELDS`, `PENDING_UPDATE_TTL`, `SprintTasksState` |
| `sprintSelection.ts` | Task selection state — selected task ID, multi-select set, drawer open state. | `useSprintSelection`, `selectSelectedTaskId`, `selectSelectedTaskIds`, `selectLogDrawerTaskId`, `selectDrawerOpen`, `selectSpecPanelOpen`, `selectIsTaskSelected` |
| `sprintEvents.ts` | Per-agent event accumulation store. Events are stored in fixed-size `RingBuffer<AnyTaskEvent>` instances (`MAX_EVENTS_PER_AGENT = 500`) — no array allocation on every push. `readAgentEvents(state, agentId)` returns events in insertion order. `selectLatestEvent(agentId)` returns the most recent event without reading the full buffer. `latestEventForTask(state, taskId)` is a pure ring-buffer lookup (no hook overhead) used outside React to find the latest event for a given task. | `useSprintEvents`, `selectLatestEvent`, `latestEventForTask`, `readAgentEvents`, `MAX_EVENTS_PER_AGENT`, `AnyTaskEvent` |
| `sprintFilters.ts` | Sprint pipeline filter state — status, repo, tag, search query. `StatusFilter` values: `all`, `backlog`, `todo`, `blocked`, `in-progress`, `review`, `open-prs`, `done`, `failed`. | `useSprintFilters`, `selectStatusFilter`, `selectRepoFilter`, `selectTagFilter`, `selectSearchQuery`, `StatusFilter` |
| `sprintUI.ts` | Sprint pipeline UI state — drawer visibility, display density, generating task IDs, orphan recovery banner. **EP-3:** `orphanRecoveryBanner` + `setOrphanRecoveryBanner` + `selectOrphanRecoveryBanner` added. | `useSprintUI`, `selectDoneViewOpen`, `selectConflictDrawerOpen`, `selectHealthCheckDrawerOpen`, `selectQuickCreateOpen`, `selectPipelineDensity`, `selectGeneratingIds`, `selectIsGenerating`, `selectOrphanRecoveryBanner`, `PipelineDensity`, `OrphanRecoveryBanner` |
| `taskWorkbench.ts` | Task creation and editing form state (title, repo, spec, dependencies, priority). Defaults to empty repo string (user must select). Validation state extracted to `taskWorkbenchValidation.ts`. | `useTaskWorkbenchStore`, `selectIsDirty` |
| `notifications.ts` | Persistent notification history — last 50 critical events, persisted to localStorage. Supports add, mark-as-read (single + all), and clear (single + all). | `useNotificationsStore`, `selectUnreadCount`, `Notification`, `NotificationType` |
| `healthCheck.ts` | Tracks stuck task IDs and dismissed IDs for the health-check overlay. Cross-store derived view (`useVisibleStuckTasks`) lives in `hooks/useVisibleStuckTasks.ts`. | `useHealthCheckStore` |
| `taskWorkbenchValidation.ts` | Validation check results for the Task Workbench (structural, semantic, operational). Extracted from `taskWorkbench.ts` for cohesion. | `useTaskWorkbenchValidation` |
| `taskWorkbenchModal.ts` | Open/close + edit-target state for the Task Workbench modal shell. `openForCreate({ groupId? })` resets the form and applies an optional epic preset; `openForCreateWithDefaults({ title?, spec?, groupId? })` resets the form and pre-populates the listed fields (used by Planner action cards' "Edit first" flow); `openForEdit(task)` loads the task into the form store. Form state itself lives in `taskWorkbench.ts`. | `useTaskWorkbenchModalStore`, `TaskWorkbenchDefaults` |
| `ide.ts` | IDE editor state — open tabs, root path, expanded directories, font/editor settings, and V2 `uiState` sub-object (activity rail, insight rail, terminal/sidebar open flags). `uiState` persists to `ide.state` alongside existing fields. `DEFAULT_UI_STATE` and `DEFAULT_INSIGHT_SECTIONS` exported for use by the restoration hook to fill new-key gaps on saved state. | `useIDEStore`, `EditorTab`, `IDEUIState`, `IDEActivity`, `InsightSectionKey`, `DEFAULT_UI_STATE`, `DEFAULT_INSIGHT_SECTIONS` |
| `ideFileCache.ts` | File content and loading-state cache for the IDE editor. Extracted from `ide.ts` for cohesion. | `useIDEFileCache` |
| `settingsNav.ts` | Settings navigation store — tracks active section for deep linking. 7 section IDs: `connections`, `repositories`, `agents`, `templates`, `memory`, `appearance`, `about`. | `useSettingsNavStore`, `SettingsSectionId` |
| `featureFlags.ts` | localStorage-backed feature flags. Defaults: all five flags `true` — V2 is production. `loadFlags()` validates each flag: `typeof parsed === 'object' && parsed !== null` guard runs before any key access; non-boolean values fall back to the per-flag default; malformed JSON or non-object values return all defaults. Flags exist for emergency rollback only. | `useFeatureFlags` |
| `floatingAgent.ts` | Floating FLEET Advisor state — session ID, messages, token estimate, open/close. Persists to `fleet:floating-agent` localStorage. Resets on 24h inactivity. Token trim drops oldest messages above 50k token estimate. | `useFloatingAgentStore`, `FloatingAgentMessage` |
| `localAgents.ts` | Local adhoc/assistant agent store — tracks spawned agents, log polling, and the in-flight spawn state. `spawnAgent()` takes only `task`, `repoPath`, and `assistant`; there is no model parameter because the main process resolves the runtime model from `agents.backendConfig`. | `useLocalAgentsStore`, `LocalAgentProcess`, `SpawnedAgent` |
| `codeReview.ts` | Code Review station state — selected task, active tab, diff files, commits, loading flags, batch IDs, review summary. `diffRefreshKey` is a monotonic counter that forces `useReviewChanges` to re-fetch the diff when incremented (e.g. after manually restoring a missing worktree). `retryDiff()` increments the key and clears the cached error. **dogfood-fix-5** | `useCodeReviewStore`, `DiffFile`, `ReviewCommit`, `ReviewTab`, `DiffMode` |
| `agentEvents.ts` | Live agent-event stream cache, capped at 2000 events per agent. `init()` subscribes to `agent:event:batch` broadcasts; subscription failure is caught, logged, and surfaced as a toast. `loadHistory(agentId)` fetches persisted events via IPC with try/catch — failure records the error message in `historyLoadErrors[agentId]` so callers can distinguish "load failed" from "no events". Both `init()` and `loadHistory()` use two-pointer ordered merge (O(n+m)). `mergeHistoryWithLiveEvents` is exported for unit-testing. | `useAgentEventsStore`, `useAgentEvents`, `mergeHistoryWithLiveEvents` |
| `reviewPartner.ts` | AI Review Partner state — review results, conversation messages, active stream IDs, and panel open/close. Messages persisted to `fleet:review-partner-messages` (localStorage, max 100 per task, 20 tasks). `invalidate(taskId)` resets a task's result to `idle`, clears messages, and nulls the active stream — called by `useAutoReview` when a stale result is detected on re-entry to `review` status. | `useReviewPartnerStore`, `ReviewState`, `ReviewPartnerStore` |
| `preflightStore.ts` | Queue of pending pre-flight warnings from the main process. Each `PreflightWarning` carries `missing` (binary names) and `missingEnvVars` (env var names from `.npmrc`). `enqueue` adds to the tail; `dequeue` removes the head after user responds. `PreflightWarningModal` consumes the head of the queue. | `usePreflightStore` |
| `taskGroups.ts` | Task group (epic) store — CRUD, optimistic updates, task membership, batch queuing, dependency management, and template instantiation. `importPlan(repo)` calls `planner:import` IPC, refreshes groups, and selects the newly imported group — `PlannerViewV2` routes through this instead of calling `window.api` directly. | `useTaskGroups`, `ImportPlanResult` |
| `dashboardData.ts` | Dashboard metrics store — throughput, activity feed, success trend, load average, and PR count. `fetchAll()` resets `loading: true` on every call (including background refreshes) so the status bar reflects in-flight state. `fetchLoad()` polls system load independently. Card-level errors tracked in `cardErrors` keyed by card name. `DailySuccessRate` imported from shared ipc-channels (no local redeclaration). | `useDashboardDataStore` |
| `prGroups.ts` | PR Group store — CRUD for stacked-PR groups. `buildGroup(id)` tracks in-flight build state in `buildingGroupIds` and reloads on success. `selectGroupsForRepo(groups, repo)` and `selectUnassignedApprovedTasks(tasks, groups, repo)` are pure selector helpers exported for direct use. | `usePrGroupsStore`, `selectGroupsForRepo`, `selectUnassignedApprovedTasks` |
