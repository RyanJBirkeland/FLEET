# Shared

Types, IPC channel constants, and utilities shared across main and renderer processes.
Source: `src/shared/`

| Module | Purpose | Key Exports |
|--------|---------|-------------|
| `task-state-machine.ts` | Canonical task state machine. `TASK_STATUSES` / `TaskStatus`, status sets (`TERMINAL_STATUSES`, `FAILURE_STATUSES`, `HARD_SATISFIED_STATUSES`, `DEPENDENCY_TRIGGER_STATUSES`) and `VALID_TRANSITIONS` are typed on `TaskStatus` — adding a new status causes a compile error until the transitions map is extended. Each terminal failure (`cancelled`, `failed`, `error`) allows a manual `→ done` transition so a human can mark a task resolved after implementing it outside the pipeline. Predicates (`isValidTransition`, `isTerminal`, `isFailure`, `isHardSatisfied`, `validateTransition`) take `TaskStatus` parameters. `isTaskStatus(value: string): value is TaskStatus` narrows at trust boundaries (DB rows, IPC payloads). **PR Builder:** `approved` status added between `review` and `done` — a human-blessed state where code is accepted and waiting to ship. `approved` satisfies hard dependencies (both `done` and `approved` are in `HARD_SATISFIED_STATUSES`) so downstream tasks can unblock before the PR merges. `review → approved` added to VALID_TRANSITIONS; `approved → done/queued/cancelled/failed` are the exit paths. `approved` is NOT terminal — the Sprint PR Poller drives `approved → done` when the PR merges. `DEPENDENCY_TRIGGER_STATUSES` is the union of `TERMINAL_STATUSES` + `'approved'` — use this set (not `TERMINAL_STATUSES`) when deciding whether to run dependency resolution. **EP-1:** `queued → done` is permitted — the auto-complete edge for when agent-manager detects that matching work already landed on `origin/main` out-of-band. **Orphan-race fix:** `queued → failed` and `queued → error` are also permitted. **#708:** `queued → backlog` added; `cancelled → queued` and `cancelled → backlog` added — revival transitions. | `TaskStatus`, `TASK_STATUSES`, `TERMINAL_STATUSES`, `FAILURE_STATUSES`, `HARD_SATISFIED_STATUSES`, `DEPENDENCY_TRIGGER_STATUSES`, `VALID_TRANSITIONS`, `isValidTransition`, `isTerminal`, `isFailure`, `isHardSatisfied`, `isTaskStatus`, `validateTransition`, `ValidationResult` |
| `task-statuses.ts` | Single source of truth for task status constants, predicates, and UI metadata. Re-exports everything from `task-state-machine.ts` (including `DEPENDENCY_TRIGGER_STATUSES`) plus `ALL_TASK_STATUSES`, `TASK_STATUS` (object-style constants), `STATUS_METADATA`, `BucketKey`, and `StatusMetadata`. `TASK_STATUS.APPROVED` added. `BucketKey` now includes `'approved'` bucket. `STATUS_METADATA` entry for `approved`: label "Approved", bucketKey `'approved'`, colorToken `'--fleet-status-done'`, icon `CheckCircle2`, actionable `true`. `TASK_STATUS` is also re-exported from `constants.ts` for callers that prefer that import path. | `ALL_TASK_STATUSES`, `TASK_STATUS`, `TaskStatus`, `TERMINAL_STATUSES`, `FAILURE_STATUSES`, `DEPENDENCY_TRIGGER_STATUSES`, `STATUS_METADATA`, `BucketKey`, `isTerminal`, `isFailure` |

| `types/git-types.ts` | Git and PR domain types. `CheckStatus` union: `'pending' \| 'pass' \| 'fail' \| 'unknown'` (`'unknown'` indicates a failed check-run fetch). `PrListPayload` carries `prs`, `checks`, and optional `repoErrors`. | `OpenPr`, `CheckStatus`, `CheckRunSummary`, `PrListPayload`, `PrReview`, `PrComment`, `PrIssueComment` |
| `types/task-types.ts` | Core task domain types — `SprintTask`, `TaskStatus`, `FailureReason`, narrow view aliases, `BatchImportTask`, and `PrGroup`. `SprintTaskPatch` is the compile-time constraint on `sprint:update` patch payloads — a `Partial<Pick<SprintTask, ...>>` over the `GENERAL_PATCH_FIELDS` field set. `SprintTask.promoted_to_review_at` is stamped by both review-transition paths for the Code Review nav badge watermark. `SprintTask.orphan_recovery_count` tracks rescue cycles; tasks at `MAX_ORPHAN_RECOVERY_COUNT` transition to `error`. `TaskGroup.is_paused` gates drain-loop claiming — when true, queued tasks belonging to that epic are skipped. **Epic-1/T-22:** `FailureReason` now includes `'git-precondition-failed'`. **UX:** `SprintTask.last_rendered_prompt` (optional TEXT) stores the exact SDK prompt for the most recent agent run; `ReviewDiffSnapshot.branchTip` (optional string) stores the HEAD SHA at snapshot capture time for incremental diff support in Code Review. **PR Builder:** `SprintTask.stacked_on_task_id` (string | null) identifies the upstream task whose branch this task is stacked on within a PR Group. `PrGroup` interface models the `pr_groups` table (status: `composing | building | open | merged`, `task_order: string[]`). | `SprintTask`, `TaskStatus`, `FailureReason`, `SprintTaskCore`, `SprintTaskSpec`, `SprintTaskExecution`, `SprintTaskPR`, `SprintTaskPatch`, `BatchImportTask`, `TaskGroup`, `EpicDependency`, `ReviewDiffSnapshot`, `PrGroup` |

| `ipc-channels/broadcast-channels.ts` | Type-safe registry of main→renderer broadcast channels. Exposes `BroadcastChannels` and the `AgentManagerDrainPausedEvent` payload emitted when the drain loop pauses after classifying a per-task failure as environmental (main-repo dirty, auth missing, network). **EP-3:** `orphan:recovered` channel added with payload `{ recovered: string[], exhausted: string[] }`. **#704:** `agent:preflightWarning` payload extended with `missingEnvVars: string[]` alongside `missing: string[]`. **AU-2:** `updates:status` broadcast added for auto-updater state changes. **UX:** `settings:externalChange` (void) broadcast fired by the DB file watcher alongside `sprint:externalChange` so renderers can re-fetch settings after out-of-band SQLite edits. | `BroadcastChannels`, `AgentManagerDrainPausedEvent` |
| `ipc-channels/update-channels.ts` | Auto-updater request/reply IPC channels — `updates:checkForUpdates` (trigger check from renderer) and `updates:install` (quit and install). Part of the auto-update feature (AU-*) for seamless version upgrades. | `UpdateChannels` |
| `ipc-channels/sprint-channels.ts` | Sprint task, review, template, group, synthesizer, planner, and PR group IPC channel type definitions. `sprint:update` `args` is typed as `[id: string, patch: SprintTaskPatch]` — narrows the patch to `GENERAL_PATCH_FIELDS` at compile time (T-63). `ReviewChannels` includes `review:shipIt` (single-task ship), `review:shipBatch` (multi-task ship with one terminal push), `review:markShippedOutsideFleet`, and `review:approveTask` (transitions task to `approved` status). `SprintChannels` also exposes operator escape-hatches `sprint:forceFailTask`, `sprint:forceDoneTask`, and `sprint:forceReleaseClaim`. **PE-3:** `groups:update` patch type extended with `is_paused?: boolean` so the preload bridge accepts pause/resume mutations. **T-17:** `sprint:create` args now include `depends_on?` field. **UX:** adds `tasks:getLastPrompt` channel (`args: [taskId]`, `result: { prompt: string | null }`) for exposing the last rendered agent prompt. **PR Builder:** adds `PrGroupChannels` interface with `prGroups:list`, `prGroups:create`, `prGroups:update`, `prGroups:addTask`, `prGroups:removeTask`, `prGroups:build`, and `prGroups:delete` channels for stacked-PR group management. | `SprintChannels`, `ReviewChannels`, `TemplateChannels`, `SynthesizerChannels`, `GroupChannels`, `ReviewPartnerChannels`, `PlannerChannels`, `PrGroupChannels` |
| `types/agent-types.ts` | Agent lifecycle, event, and status types. `AgentEvent` discriminated union covers all event types including `agent:playground` (with `contentType: PlaygroundContentType`). `PlaygroundContentType` is the canonical definition (`'html' \| 'svg' \| 'markdown' \| 'json'`); `PLAYGROUND_CONTENT_TYPE_LABELS` maps each value to a display label. Both re-exported via `types/index.ts`. `SpawnLocalAgentArgs` carries only `task`, `repoPath`, and `assistant?` — the main process resolves the model from `agents.backendConfig`, so IPC callers cannot override it. **T-20:** `agent:error` variant carries optional `taskId`, `messagesConsumed`, and `lastEventType` for stream-error forensics. | `AgentEvent`, `AgentType`, `PlaygroundContentType`, `PLAYGROUND_CONTENT_TYPE_LABELS`, `AgentMeta`, `AgentManagerStatus`, `SpawnLocalAgentArgs` |
| `types/agent-prompt.ts` | Cross-cutting `BuildPromptInput` consumed by `lib/prompt-composer.ts` and the per-agent builders under `agent-manager/prompt-*.ts`. | `time.ts` | Timestamp utilities shared across main and renderer. Exports `nowIso()` (current timestamp in ISO 8601 format) and duration constants `MS_PER_HOUR = 3_600_000` and `MS_PER_DAY = 86_400_000`. Data modules import these instead of scattering inline millisecond arithmetic. | `nowIso`, `MS_PER_HOUR`, `MS_PER_DAY` |
| `sanitize-tags.ts` | Coerces the SQLite `tags` column to `string[] \| null`. Accepts JSON-encoded arrays and legacy comma-separated strings (seeded via direct SQL inserts that bypass JSON.stringify). Drops empty entries and returns `null` for unusable input. | `sanitizeTags` |
| `sanitize-epic-depends-on.ts` | Coerces the SQLite `task_groups.depends_on` column to a validated `EpicDependency[]`. Accepts JSON strings, arrays, or nullish input; filters entries that lack a non-empty `id` or whose `condition` is not `on_success`/`always`/`manual`; returns `[]` for malformed JSON or unsupported value types rather than throwing. | `sanitizeEpicDependsOn` |
| `types/backend-settings.ts` | Backend-selection types shared across main and renderer. `BackendKind = 'claude' \| 'opencode'`. `BackendSettings` carries per-agent-type `AgentBackendConfig` (backend + model) and `opencodeExecutable` (path to the opencode CLI; default `'opencode'` for PATH lookup). | `BackendKind`, `AgentBackendConfig`, `BackendSettings` |
| `types/revision.ts` | Structured revision feedback types for pipeline retry prompts. `RevisionFeedback` carries a human-readable `summary` and an array of `RevisionDiagnostic` entries (each with `file`, optional `line`, `kind`, `message`, and optional `suggestedFix`). Serialised as JSON into `sprint_tasks.notes` when pre-review verification fails so retry agents receive machine-readable diagnostics instead of raw stderr. Includes `parseRevisionFeedback()` for deserializing notes and `isRevisionFeedback()` type guard. Re-exported via `types/index.ts`. | `RevisionFeedback`, `RevisionDiagnostic`, `parseRevisionFeedback` |
