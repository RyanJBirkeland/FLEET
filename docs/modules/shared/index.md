# Shared

Types, IPC channel constants, and utilities shared across main and renderer processes.
Source: `src/shared/`

| Module | Purpose | Key Exports |
|--------|---------|-------------|
| `task-state-machine.ts` | Canonical task state machine. `TASK_STATUSES` / `TaskStatus`, status sets (`TERMINAL_STATUSES`, `FAILURE_STATUSES`, `HARD_SATISFIED_STATUSES`) and `VALID_TRANSITIONS` are typed on `TaskStatus` — adding a new status causes a compile error until the transitions map is extended. Predicates (`isValidTransition`, `isTerminal`, `isFailure`, `isHardSatisfied`, `validateTransition`) take `TaskStatus` parameters. `isTaskStatus(value: string): value is TaskStatus` narrows at trust boundaries (DB rows, IPC payloads). | `TaskStatus`, `TASK_STATUSES`, `TERMINAL_STATUSES`, `FAILURE_STATUSES`, `HARD_SATISFIED_STATUSES`, `VALID_TRANSITIONS`, `isValidTransition`, `isTerminal`, `isFailure`, `isHardSatisfied`, `isTaskStatus`, `validateTransition`, `ValidationResult` |
| `task-statuses.ts` | Single source of truth for task status constants, predicates, and UI metadata. Re-exports everything from `task-state-machine.ts` plus `ALL_TASK_STATUSES`, `TASK_STATUS` (object-style constants), `STATUS_METADATA`, `BucketKey`, and `StatusMetadata`. `TASK_STATUS` is also re-exported from `constants.ts` for callers that prefer that import path. | `ALL_TASK_STATUSES`, `TASK_STATUS`, `TaskStatus`, `TERMINAL_STATUSES`, `FAILURE_STATUSES`, `STATUS_METADATA`, `BucketKey`, `isTerminal`, `isFailure` |
| `types/review-types.ts` | Review Partner domain types — findings, inline comments, review results, chat wire types. Relocated from `src/shared/review-types.ts`; re-exported via `src/shared/types/index.ts`. | `FindingSeverity`, `FindingCategory`, `InlineComment`, `FileFinding`, `ReviewFindings`, `ReviewResult`, `PartnerMessage`, `ChatChunk` |
| `types/task-types.ts` | Core task domain types — `SprintTask`, `TaskStatus`, `FailureReason` union (auth, timeout, test_failure, compilation, spawn, no_commits, unknown), and narrow view aliases (`SprintTaskCore`, `SprintTaskSpec`, `SprintTaskExecution`, `SprintTaskPR`). | `SprintTask`, `TaskStatus`, `FailureReason`, `SprintTaskCore`, `SprintTaskSpec`, `SprintTaskExecution`, `SprintTaskPR` |
| `spec-quality/types.ts` | Shared spec validation types — `IssueCode` union (includes `PRESCRIPTIVENESS_CHECK_FAILED`), `IssueSeverity`, `SpecIssue`, `ParsedSpec`, `ParsedSection`, `SpecQualityResult` | `IssueCode`, `IssueSeverity`, `SpecIssue`, `ParsedSpec`, `ParsedSection`, `SpecQualityResult` |
| `ipc-channels/settings-channels.ts` | Settings IPC channel type definitions including `settings:hasSecret` (returns boolean, avoids sending sensitive plaintext to renderer) | `SettingsChannels`, `ClaudeConfigChannels`, `AuthChannels`, `OnboardingChannels` |
| `ipc-channels/broadcast-channels.ts` | Type-safe registry of main→renderer broadcast channels. Exposes `BroadcastChannels` and the `AgentManagerDrainPausedEvent` payload emitted when the drain loop pauses after classifying a per-task failure as environmental (main-repo dirty, auth missing, network). | `BroadcastChannels`, `AgentManagerDrainPausedEvent` |
| `ipc-channels/sprint-channels.ts` | Sprint task, review, template, group, synthesizer, and planner IPC channel type definitions. `ReviewChannels` includes `review:shipIt` (single-task ship) and `review:shipBatch` (multi-task ship with one terminal push — returns `shippedTaskIds` on success, `failedTaskId` + `shippedTaskIds` on failure). `SprintChannels` also exposes operator escape-hatches `sprint:forceFailTask` and `sprint:forceDoneTask` (payload `{ taskId, reason?, force? }`, result `{ ok: true }`) that bypass state-machine transitions. | `SprintChannels`, `ReviewChannels`, `TemplateChannels`, `SynthesizerChannels`, `GroupChannels`, `ReviewPartnerChannels`, `PlannerChannels` |
| `types/agent-types.ts` | Agent lifecycle, event, and status types. `AgentEvent` discriminated union covers all event types including `agent:playground` (with `contentType: PlaygroundContentType`). `PlaygroundContentType` is the canonical definition (`'html' \| 'svg' \| 'markdown' \| 'json'`); `PLAYGROUND_CONTENT_TYPE_LABELS` maps each value to a display label. Both re-exported via `types/index.ts`. `SpawnLocalAgentArgs` carries only `task`, `repoPath`, and `assistant?` — the main process resolves the model from `agents.backendConfig`, so IPC callers cannot override it. | `AgentEvent`, `AgentType`, `PlaygroundContentType`, `PLAYGROUND_CONTENT_TYPE_LABELS`, `AgentMeta`, `AgentManagerStatus`, `SpawnLocalAgentArgs` |
| `types/agent-prompt.ts` | Cross-cutting `BuildPromptInput` consumed by `lib/prompt-composer.ts` and the per-agent builders under `agent-manager/prompt-*.ts`. T-21 + T-22 (Phase 2) moved this here so the per-agent builders no longer back-import from `lib/`, breaking the previous import cycle. | `BuildPromptInput` |
| `sanitize-tags.ts` | Coerces the SQLite `tags` column to `string[] \| null`. Accepts JSON-encoded arrays and legacy comma-separated strings (seeded via direct SQL inserts that bypass JSON.stringify). Drops empty entries and returns `null` for unusable input. | `sanitizeTags` |
| `sanitize-epic-depends-on.ts` | Coerces the SQLite `task_groups.depends_on` column to a validated `EpicDependency[]`. Accepts JSON strings, arrays, or nullish input; filters entries that lack a non-empty `id` or whose `condition` is not `on_success`/`always`/`manual`; returns `[]` for malformed JSON or unsupported value types rather than throwing. | `sanitizeEpicDependsOn` |
| `types/backend-settings.ts` | Backend-selection types shared across main and renderer. `BackendKind = 'claude' \| 'local' \| 'opencode'`. `BackendSettings` carries per-agent-type `AgentBackendConfig` (backend + model), a `localEndpoint` URL for the local backend, and `opencodeExecutable` (path to the opencode CLI; default `'opencode'` for PATH lookup). | `BackendKind`, `AgentBackendConfig`, `BackendSettings` |
| `types/revision.ts` | Structured revision feedback types for pipeline retry prompts. `RevisionFeedback` carries a human-readable `summary` and an array of `RevisionDiagnostic` entries (each with `file`, optional `line`, `kind`, `message`, and optional `suggestedFix`). Serialised as JSON into `sprint_tasks.notes` when pre-review verification fails so retry agents receive machine-readable diagnostics instead of raw stderr. Re-exported via `types/index.ts`. | `RevisionFeedback`, `RevisionDiagnostic` |
